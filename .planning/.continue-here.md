## Critical Anti-Patterns (do NOT repeat these)
- **Cache clearing incomplete**: SpotOn uses `spoton.db` (own cache) AND LMS's `cache.db`. Always clear BOTH when debugging cache issues.
- **Stale SHA after re-tag**: When a git tag is deleted and re-created, CI builds a new zip with a different SHA. Always `rm` the local zip and re-download before computing SHA for repo.xml.
