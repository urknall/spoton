# Requirements: SpotOn — v1.0 Archive

**Defined:** 2026-05-26
**Completed:** 2026-06-03
**Core Value:** Reliable Spotify playback and Connect integration on LMS — Browse, stream, and control via Spotify app, without 429 bursts, zombie daemons, or audio glitches.

## v1 Requirements (all complete)

### Authentication

- [x] **AUTH-01**: Plugin obtains Spotify access token via OAuth 2.0 Authorization Code + PKCE browser flow through per-user Spotify Developer App
- [x] **AUTH-02**: Access token is cached and automatically refreshed before expiry
- [x] **AUTH-03**: Connect daemons are proactively restarted at 50-minute uptime to prevent silent token expiry
- [x] **AUTH-04**: Credential storage uses LMS Prefs with restricted plugin namespace (PKCE replaced filesystem credential files)
- [x] **AUTH-05**: Multiple Spotify accounts can be configured per LMS instance
- [x] **AUTH-06**: Account switching is available in the plugin menu

### Browse & Navigation

- [x] **NAV-01**: Top-level menu structure: Home, Search, Library
- [x] **NAV-02**: Home feed shows Recently Played, Made For You mixes (via category ID trick), Top Tracks
- [x] **NAV-03**: Library shows Liked Songs, Saved Albums, Followed Artists, User Playlists as sub-items
- [x] **NAV-04**: Search supports free-text with results categorized (Tracks, Albums, Artists, Playlists)
- [x] **NAV-05**: Artist detail page shows Discography (Albums, Singles, Compilations)
- [x] **NAV-06**: Album detail page shows tracklist with track number, duration, featuring artists
- [x] **NAV-07**: Playlist detail page shows paginated tracks, description, creator
- [x] **NAV-08**: Liked Songs are exposed unconditionally (no gating behind custom Client ID)
- [x] **NAV-09**: Library items sortable (recently added as default)
- [x] **NAV-10**: Endpoints removed in Dev Mode (Artist Top Tracks, Browse Categories, New Releases, Related Artists) are gracefully hidden, not errored
- [x] **NAV-11**: Search pagination handles limit=10 per request (Dev Mode constraint)

### Audio Streaming

- [x] **STR-01**: Single-track playback via librespot `--single-track` mode writing PCM to stdout
- [x] **STR-02**: FLAC transcoding pipeline as default (`son → flc` via custom-convert.conf)
- [x] **STR-03**: PCM passthrough pipeline for capable players
- [x] **STR-04**: MP3 transcoding pipeline as legacy fallback
- [x] **STR-05**: OGG-Direct passthrough for players that support OGG natively
- [x] **STR-06**: Bitrate selection (96/160/320 kbps) configurable per plugin settings + per-player override
- [x] **STR-07**: Seeking via `--start-position` parameter in transcoding pipeline
- [x] **STR-08**: Volume normalization (Replay Gain) optional, globally configurable
- [x] **STR-09**: Gapless playback between consecutive tracks
- [x] **STR-10**: Hourly cleanup of orphaned librespot processes
- [x] **STR-11**: Audio cache disabled (--disable-audio-cache hardcoded)

### Spotify Connect

- [x] **CON-01**: One librespot Connect daemon per LMS player
- [x] **CON-02**: Daemon lifecycle management (start at init, stop at shutdown, restart on crash with backoff)
- [x] **CON-03**: Event dispatching from binary to LMS via HTTP /event endpoint (start/stop/change/volume/pause)
- [x] **CON-04**: Transfer playback from Spotify app to LMS player starts audio within 3 seconds
- [x] **CON-05**: Play/Pause/Skip/Volume controllable from Spotify app
- [x] **CON-06**: Sync-group handling: one daemon on master player, name = concatenated player names
- [x] **CON-07**: mDNS/ZeroConf discovery for Connect receivers (optionally disableable per player)
- [x] **CON-08**: Mutual exclusion between Browse-streaming and Connect sessions
- [x] **CON-09**: Connect daemon PIDs excluded from LMS `killHangingProcesses` guard
- [x] **CON-10**: Connect per player enable/disable in settings
- [x] **CON-11**: Volume suppression window after Connect start (prevents volume jumps)
- [x] **CON-12**: HTTP-streaming audio transport (not FIFO)
- [x] **CON-13**: Position sync via `startOffset` (never `['time', N]` in stream mode)
- [x] **CON-14**: Sink-level rate-limiting (wall-clock speed, nanosecond-accurate)
- [x] **CON-15**: Differential daemon restart on sync-group changes (only affected daemons)
- [x] **CON-16**: Unique port assignment per daemon (HTTP audio + ZeroConf) from Manager pool
- [x] **CON-17**: Progress stored in `pluginData` before `playlist play` command (race condition prevention)

### API Infrastructure

- [x] **API-01**: Central HTTP client (`API/Client.pm`) as sole HTTP egress point
- [x] **API-02**: Rate limiting via sliding window with dual-token pressure distribution
- [x] **API-03**: Response caching with domain-specific TTLs (60s Library, 300s Browse, 3600s Metadata)
- [x] **API-04**: `Retry-After` header respected on 429 responses
- [x] **API-05**: Batch API endpoints removed in Dev Mode — individual fetches with throttle
- [x] **API-06**: All HTTP calls use `SimpleAsyncHTTP` with explicit timeouts (never blocking)

### LMS Integration

- [x] **LMS-01**: `spotify://` URI protocol handler registered and functional
- [x] **LMS-02**: Web-based settings UI under LMS Settings
- [x] **LMS-03**: i18n support (11 languages: CS, DA, DE, EN, ES, FR, IT, NL, NO, PL, SV) via LMS strings mechanism
- [x] **LMS-04**: install.xml manifest with correct metadata, minVersion 8.0, version 1.0.0
- [x] **LMS-05**: custom-convert.conf with `son → pcm/flc/mp3/ogg` transcoding pipelines
- [x] **LMS-06**: x86_64 binary present (multi-arch deferred to v1.1)
- [x] **LMS-07**: Binary capability detection via `--check` JSON with version enforcement
- [x] **LMS-08**: Player-specific preferences (bitrate override, format dropdown, Connect on/off)
- [x] **LMS-09**: Don't Stop The Music integration for auto-play after playlist end (Browse mode)
- [x] **LMS-10**: Custom binary support (user-provided binary override via spoton-custom)
- [x] **LMS-11**: Transcoding table updated per-track via formatOverride (not globally)

## v2 Requirements (unchanged, carried forward)

- **WB-01**: Like/Save/Follow actions via unified `PUT /me/library` endpoint
- **WB-02**: Artist Radio / Track Radio via `recommendations` endpoint

### Audio Transport Upgrade

- **AT-01**: HTTP-streaming as primary Connect audio transport (replaces FIFO)
- **AT-02**: Clean seek in Connect mode via HTTP Content-Range
- **AT-03**: No white noise on reconnect (proper connection semantics)

### Extended Features

- **EXT-01**: Online Library Importer (Spotify library → LMS database for local search)
- **EXT-02**: Podcast & Audiobook support
- **EXT-03**: Browse Categories and New Releases (requires Extended Quota or workaround)
- **EXT-04**: "Fans also like" / Related Artists (requires Extended Quota)
