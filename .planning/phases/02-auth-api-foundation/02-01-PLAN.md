<objective>
Extend the librespot-spoton Rust binary with `--authenticate` and `--get-token` subcommands backed by librespot-core.

Purpose: The Perl plugin cannot authenticate directly with Spotify (login5 protocol requires native code). The binary bridges this gap: `--authenticate` acquires credentials from username/password, `--get-token` retrieves a Web API access token from cached credentials. Both are short-lived processes per D-03.

Output: Updated binary that satisfies the AUTH-01 contract (token acquisition via Keymaster/login5) and writes credentials in librespot-native format per D-02.
</objective>

    Extend main.rs arg parsing:
    - Add a Mode enum: Check, Authenticate, GetToken (Connect is Phase 5, print error and exit).
    - Add CLI flags: --authenticate, --get-token, --username/-u (takes next arg), --password/-p (takes next arg), --cache/-c (takes next arg), --scope (takes next arg), --client-id (takes next arg).
    - Keep existing --check and -n/--name handling unchanged.
    - Dispatch on Mode after arg parsing.

    For Mode::Authenticate:
    - Validate that username, password, and cache_dir are provided (exit 1 with usage error to stderr if missing).
    - Use librespot-core's SessionConfig and Session to connect to Spotify AP (access point).
    - Authenticate using Credentials::with_password(username, password).
    - On success: the session stores credentials in the cache directory (librespot-core's Cache handles this natively when constructed with a cache path). Print "authorized" to stdout. Exit 0.
    - On failure: print error to stderr, exit 1.

    For Mode::GetToken:
    - Validate that cache_dir is provided.
    - Construct librespot-core's Cache from cache_dir. Load stored credentials via cache.credentials().
    - Create a Session using the cached credentials.
    - Use session.token_provider().get_token(scopes) to obtain a Mercury/Keymaster token (the scopes string should be split on comma).
    - Print JSON to stdout: {"accessToken":"<token>","expiresIn":<seconds>} using serde_json.
    - On failure: print error to stderr, exit 1.

    Both modes run inside a tokio runtime (#[tokio::main] or Runtime::new().block_on()).

    Preserve the existing --check behavior exactly (do not modify the check output format or the regex contract).

    IMPORTANT: Do NOT add librespot-oauth to Cargo.toml in Phase 2. OAuth browser redirect is out of scope for this phase. Phase 2 uses login5 username/password authentication only. The --enable-oauth flag is a future enhancement and must not be implemented here.

    IMPORTANT: The librespot-core API may have changed between versions. Use cargo doc --open or check the librespot-core source for the exact API surface. Key types to look for: SessionConfig, Session, Cache, Credentials, TokenProvider. If the token_provider API is not available in the crate version, use the Mercury protocol directly (hm://keymaster/token/authenticated?client_id=...&scope=...).
