<objective>
Create the TokenManager module that bridges the librespot binary and the Perl plugin for token lifecycle management.

Purpose: Every Spotify API call requires a valid access token. TokenManager handles: (1) spawning the binary with `--get-token` to acquire tokens, (2) caching tokens with pre-expiry TTL, (3) scheduling proactive refresh via LMS timers, (4) managing credential directories with secure permissions, and (5) supporting multiple accounts with separate cache subdirectories.

Output: `API/TokenManager.pm` with full token lifecycle management plus a comprehensive test file using mock binary and LMS stubs.
</objective>

    - Module vars: $log, $prefs, $cache (same pattern as Helper.pm).

    Public methods (all class methods, not instance):

    1. refreshToken($class, $accountId, $cb):
       - Get binary path via Plugins::SpotOn::Helper->get().
       - Build cache dir: _cacheDir($accountId).
       - Build command with shell-safe quoting (copy pattern from Helper.pm line 67-68): spoton -n 'SpotOn' --cache <dir> --get-token --scope <scopes>
       - Wrap backtick call in alarm(10) (Pitfall 1) with eval/die timeout handler.
       - Parse stdout JSON. Look for keys accessToken (or access_token -- handle both) and expiresIn (or expires_in).
       - On success: call _cacheToken($accountId, $tokenData), invoke $cb->($token).
       - On failure: log error, invoke $cb->(undef).

    2. getToken($class, $accountId, $cb):
       - Check cache first: $cache->get("spoton_token_$accountId").
       - If cache hit: invoke $cb->($cachedToken) immediately.
       - If cache miss: call refreshToken($accountId, $cb).

    3. addAccount($class, $username, $password, $cb):
       - Validate inputs (non-empty).
       - Compute accountId: substr(Digest::MD5::md5_hex(Slim::Utils::Unicode::utf8toLatin1Transliterate($username) || $username), 0, 8).
       - Create temp cache dir via _newAccountCacheDir().
       - Build command: spoton -n 'SpotOn' --username <u> --password <p> --authenticate --cache <dir>
       - Shell-safe quote ALL user-supplied values.
       - Wrap in alarm(15) (auth takes longer than token fetch).
       - On stdout matching /^authorized/i: finalize dir (rename temp to permanent per accountId), set permissions, store account info in prefs (accounts hash: accountId => {username => $username, displayName => $username}), invoke $cb->($accountId, undef).
       - Then immediately try refreshToken to fetch display name via getMe (but this is Phase 2 Plan 03 -- for now just store username as displayName).
       - On failure: clean up temp dir, invoke $cb->(undef, "Authentication failed: $error").

    4. removeAccount($class, $accountId):
       - Remove from prefs accounts hash.
       - Remove cache token: $cache->remove("spoton_token_$accountId").
       - Remove credential directory: File::Path::rmtree(_cacheDir($accountId)).
       - If activeAccount was this account, clear it.

    5. getAccountIds($class):
       - Return keys of $prefs->get('accounts') hash.

    6. getActiveAccountName($class, $client):
       - Determine active account: $prefs->client($client)->get('activeAccount') || $prefs->get('activeAccount').
       - Look up displayName from accounts hash.
       - Return displayName or undef.

    7. refreshAllTokens($class):
       - For each accountId in getAccountIds(): call refreshToken.
       - Re-arm the timer: Slim::Utils::Timers::killTimers($class, \&refreshAllTokens) then setTimer for TOKEN_REFRESH_TIMER from now.

    Private methods:
    - _cacheDir($accountId): catdir(preferences('server')->get('cachedir'), 'spoton', $accountId) -- use File::Spec::Functions::catdir.
    - _newAccountCacheDir(): catdir(preferences('server')->get('cachedir'), 'spoton', '__AUTHENTICATE__') -- temp dir for in-progress auth.
    - _cacheToken($accountId, $tokenData): compute TTL as expiresIn - TOKEN_EXPIRY_BUFFER (min 60), cache->set("spoton_token_$accountId", $token, $ttl).
    - _setPermissions($accountId): chmod 0700 on _cacheDir, chmod 0600 on credentials.json within it.
    - _finalizeAccountDir($tempDir, $accountId): rename $tempDir to _cacheDir($accountId) using File::Copy::move or rename.

    Use main::INFOLOG guard for all info-level logging. Never log the accessToken value (T-02-03 security).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-supplied username/password -> binary CLI args | Shell injection risk via backtick |
| Binary stdout -> token parse | Malformed JSON could crash eval |
| Filesystem -> credential permissions | World-readable credentials = info disclosure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | Shell injection via username/password in --authenticate backtick | mitigate | Shell-safe quoting: s/'/'\\''/g on all user-supplied values before backtick; sprintf with single-quoted arguments |
| T-02-05 | Information Disclosure | credentials.json world-readable | mitigate | _setPermissions: chmod 0600 credentials.json, chmod 0700 parent directory, called immediately after binary writes |
| T-02-06 | Denial of Service | Binary --get-token hangs on network outage | mitigate | alarm(10) timeout wrapper around backtick call; eval/die handler kills stale process |
| T-02-07 | Information Disclosure | Token logged to server.log | mitigate | Never log accessToken value; log only account ID and expiry time |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No external packages installed; all LMS-bundled modules |
</threat_model>
