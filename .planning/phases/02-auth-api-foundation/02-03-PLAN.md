    Create Plugins/SpotOn/API/Client.pm with package Plugins::SpotOn::API::Client.

    Module-level state:
    - Constants: RATE_LIMIT_CACHE_KEY => 'spoton_rate_limit_exceeded', RATE_LIMIT_DEFAULT_BACKOFF => 5, MAX_CONCURRENT_REQUESTS => 3, API_BASE => 'https://api.spotify.com/v1', REQUEST_TIMEOUT => 30.
    - Module vars: $inflightCount = 0, $log, $prefs, $cache.
    - Export RATE_LIMIT_CACHE_KEY so Plugin.pm can read it for the OPML hint (D-12).

    Public methods:

    1. reset($class):
       - Set $inflightCount = 0. Called by Plugin.pm::initPlugin on startup to prevent Pitfall 2 (stale counter after restart).

    2. getMe($class, $accountId, $cb):
       - Call _request('get', 'me', { _accountId => $accountId, _noCache => 1 }, $cb).
       - This is the only endpoint in Phase 2 (D-15). Phase 3 will add Browse/Search/Library methods.

    3. _request($class, $method, $path, $params, $cb):
       The core request pipeline:
       a. Check rate limit flag: if $cache->get(RATE_LIMIT_CACHE_KEY), invoke $cb->(undef, { error => 'rate_limited', code => 429 }) and return.
       b. Check response cache (unless $params->{_noCache}): build cache key "spoton_resp_$path" (include query params in key if present). If cache hit, invoke $cb->($cached) and return.
       c. Check concurrency cap: if $inflightCount >= MAX_CONCURRENT_REQUESTS, defer via Slim::Utils::Timers::setTimer(undef, Time::HiRes::time() + 0.1, sub { $class->_request(...) }) and return.
       d. Increment $inflightCount.
       e. Get token via TokenManager->getToken($accountId, sub { ... }). Extract $accountId from $params->{_accountId}.
       f. If no token: decrement $inflightCount, invoke $cb->(undef, { error => 'no_token' }), return.
       g. Build URL: API_BASE . "/$path". Append query string from $params (excluding keys starting with _).
       h. Create SimpleAsyncHTTP with timeout => REQUEST_TIMEOUT, cache => 0.
       i. Call $http->$method($url, 'Authorization' => "Bearer $token", 'Accept' => 'application/json').

    4. _onSuccess($class, $http, $path, $params, $cb):
       - Decrement $inflightCount.
       - Parse JSON: eval { from_json($http->content) }.
       - On parse error: log error, invoke $cb->(undef, { error => 'parse_error' }), return.
       - Cache response with TTL from _cacheTTL($path) unless $params->{_noCache} or TTL is 0.
       - Invoke $cb->($result).

    5. _onError($class, $http, $error, $path, $params, $cb):
       - Decrement $inflightCount (CRITICAL: both success and error paths must decrement).
       - Extract status code from $http->response->code (guard against undef response).
       - If 429: read Retry-After header (default RATE_LIMIT_DEFAULT_BACKOFF). Cap at 300 seconds max (security: prevent malicious Retry-After). Set $cache->set(RATE_LIMIT_CACHE_KEY, 1, $retryAfter). Invoke $cb->(undef, { error => 'rate_limited', code => 429 }).
       - If 401: invalidate token cache for this account ($cache->remove("spoton_token_$accountId")), invoke $cb->(undef, { error => 'unauthorized', code => 401 }). (Auto-retry on 401 is a Phase 3 enhancement.)
       - Else: log error, invoke $cb->(undef, { error => $error, code => $code }).

    6. _cacheTTL($path):
       - Return 0 for paths starting with "me/player" (playback state, never cache).
       - Return 60 for paths matching "me/tracks" or "me/albums" (library items).
       - Return 3600 for paths matching "tracks/", "albums/", "artists/" (metadata).
       - Return 300 for paths matching "playlists/", "browse/" (browse data).
       - Return 0 for "me" (profile, always fresh).
       - Default: 0 (no cache for unknown paths).

    Use main::INFOLOG guard for info logging. Log errors unconditionally.
