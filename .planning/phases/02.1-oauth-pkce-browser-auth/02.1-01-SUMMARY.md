# Phase 02.1 Plan 01: TokenManager PKCE Rewrite Summary

**One-liner:** Complete rewrite of TokenManager.pm from binary Keymaster/login5 auth to async OAuth 2.0 PKCE using Digest::SHA + Crypt::OpenSSL::Random + SimpleAsyncHTTP, with 28-test unit test suite.

## What Was Built

### Task 0: REQUIREMENTS.md Update
Updated three requirement rows in section 3.4 to resolve the contradiction between PKCE being listed as "out-of-scope" while Phase 02.1 makes it the sole auth mechanism:
- **AUT-01**: Updated to reference OAuth 2.0 Authorization Code + PKCE as the sole auth mechanism; references D-04 per-user Developer App
- **AUT-04**: Marked REMOVED — login5 deactivated by Spotify (verified May 2026); references Phase 02.1 D-01/D-02
- **AUT-06**: Updated to describe per-user Developer App model with Client-ID in LMS Settings

### Task 1: TokenManager.pm Complete Rewrite

Deleted all Keymaster/login5 code:
- `addAccount()` — username/password via `--authenticate`
- `refreshToken()` blocking backtick via `--get-token`
- `_cacheDir()`, `_newAccountCacheDir()`, `_setPermissions()`, `_finalizeAccountDir()`
- `CLIENT_ID` constant (replaced by per-user `$prefs->get('clientId')`)
- `File::Copy`, `File::Path`, `File::Spec::Functions`, `Slim::Utils::Unicode` imports
- `require Plugins::SpotOn::Helper` import

Added PKCE implementation:
- `_generatePkce()` — 32 random bytes via Crypt::OpenSSL::Random, encode_base64url, sha256 challenge
- `_generateState()` — 16 random bytes, URL-safe base64, OAuth CSRF nonce
- `startOAuthFlow($clientId)` — generates PKCE pair + state, caches under `spoton_pkce_$state` (TTL=600s), builds Spotify auth URL via URI
- `_buildRedirectUri()` — always uses `http://127.0.0.1:{httpPort}/plugins/SpotOn/settings/callback` per RESEARCH.md Pitfall 5
- `exchangeCode($code, $verifier, $clientId, $redirectUri, $cb)` — async POST to SPOTIFY_TOKEN_URL, fetches user profile, stores tokens
- `refreshToken($accountId, $cb)` — async POST with `grant_type=refresh_token`, handles refresh token rotation per Pitfall 7
- `_fetchUserProfile($accessToken, $cb)` — async GET to Spotify `/me` for stable accountId generation
- `_storeTokens($accessToken, $refreshToken, $expiresIn, $userProfile, $cb)` — MD5-based accountId, access in Cache (TTL), refresh in Prefs

Preserved public API for Client.pm compatibility:
- `getToken($accountId, $cb)` — cache-first, fallback to `refreshToken`
- `getAccountIds()`, `getActiveAccountName($client)`, `removeAccount($accountId)`, `refreshAllTokens()`
- `_cacheToken($accountId, $accessToken, $expiresIn)` — signature updated (4 args, not 3 with hashref)

Security mitigations applied:
- T-02.1-01: `Crypt::OpenSSL::Random` for code_verifier (not `rand()`)
- T-02.1-02: State nonce with TTL=600s cache; consume-on-use in callback
- T-02.1-03: No log calls pass token values
- T-02.1-04: Refresh token in Prefs, not Cache
- T-02.1-05: Validates `access_token` field in response before storing
- T-02.1-06: code_verifier cached with TTL=600s

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] MockHTTP package path not loadable as separate stub file**
- **Found during:** Task 2, while debugging test failure (`Can't locate object method "content" via package "MockHTTP"`)
- **Issue:** `write_stub()` writes files using `Foo/Bar.pm` path convention. `MockHTTP` has no double-colon path, so `write_stub($stub_dir, 'MockHTTP', ...)` writes to `$stub_dir/MockHTTP.pm` — but the stub was bless-ed with `bless {}, 'MockHTTP'` and the package was never loaded via `require`
- **Fix:** Defined `MockHTTP` inline inside the `Slim::Networking::SimpleAsyncHTTP` stub file, using `Slim::Networking::MockHTTP` as the package name (avoids path issue entirely)
- **Files modified:** t/07_token_manager.t

**2. [Rule 1 - Bug] Test expectation mismatch for minimum TTL**
- **Found during:** Task 2, test 19 failure
- **Issue:** Test expected TTL=60 for `expiresIn=200`, but the actual code returns `200` (since `200 > 60`, the ternary returns `$expiresIn`, not `60`). The `60` floor only applies when `$expiresIn <= 60`.
- **Fix:** Updated test expectation to `200` with updated description. The logic is correct per the PATTERNS.md `_cacheToken` pattern.
- **Files modified:** t/07_token_manager.t

**3. [Rule 1 - Bug] preferences() not in scope inside SKIP block**
- **Found during:** Task 2, runtime error at line 385
- **Issue:** `preferences()` is installed into the caller's namespace by `Slim::Utils::Prefs->import()`. Since `unshift @INC` runs at runtime, `use Slim::Utils::Prefs` at compile time would find the real system module. Using `require Slim::Utils::Prefs; Slim::Utils::Prefs->import()` after `unshift @INC` correctly installs the stub's version.
- **Fix:** Changed `use` statements to `require` + explicit `->import()` calls after `unshift @INC`
- **Files modified:** t/07_token_manager.t
