<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Spotify -> LMS callback | Spotify redirects browser with ?code=&state= to LMS; untrusted input from URL params |
| Cache state lookup | PKCE state must match a prior startOAuthFlow; prevents CSRF and code injection |
| Callback -> TokenManager | Passes authorization code for exchange; must not log the code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.1-02 | Spoofing | OAuth state parameter | mitigate | Validate state against cache; reject if not found; consume after use ($cache->remove) to prevent replay |
| T-02.1-07 | Spoofing | CSRF on callback route | accept | State parameter IS the CSRF protection for OAuth; addPageFunction route must NOT use protectURI (would block Spotify's redirect per Pitfall 6) |
| T-02.1-08 | Tampering | Authorization code injection | mitigate | State validation ensures code is from a locally-initiated flow; foreign codes are rejected |
| T-02.1-09 | Information Disclosure | Code in URL/logs | mitigate | Never log $paramRef->{code}; log only state validation outcome |
| T-02.1-10 | Denial of Service | Stale PKCE cache entries | mitigate | TTL=600s on cache entries; entries expire automatically if flow is abandoned |
| T-02.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan |
</threat_model>
