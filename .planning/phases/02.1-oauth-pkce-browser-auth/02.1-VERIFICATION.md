### Gaps Summary

Two gaps blocking full goal achievement:

**Gap 1 (SC-2): Display name missing from callback success page.**
The ROADMAP success criterion explicitly requires the callback page to show "Erfolgreich verbunden! **with the user's display name**". The implemented `_renderResult` function is called with `(1, undef)` on success — the display name is never fetched from the stored accounts hash or passed to the HTML renderer. Fix: in the `exchangeCode` success callback in Callback.pm, read `$prefs->get('accounts')->{$accountId}{displayName}` and pass it to `_renderResult`.

**Gap 2 (.planning/REQUIREMENTS.md not updated).**
The GSD planning requirements tracking file `.planning/REQUIREMENTS.md` was not updated and still contradicts the implementation. AUTH-01 still references Keymaster/login5, and "PKCE/OAuth browser flow" and "Spotify Developer App registration" remain in the Out-of-Scope table. While the root `REQUIREMENTS.md` was correctly updated (target of Plan 01 Task 0), the `.planning/REQUIREMENTS.md` is the file the GSD toolchain and future phases will reference for requirement traceability. Fix: update `.planning/REQUIREMENTS.md` AUTH-01 description, remove PKCE/Developer App from Out-of-Scope table, update traceability rows to reference Phase 02.1.

These two gaps are independent — Gap 2 is a documentation/tracking artifact gap; Gap 1 is a functional display gap in the callback success page.
