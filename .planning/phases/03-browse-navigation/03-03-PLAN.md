    **A. _searchFeed($client, $callback, $args)** — per D-10, D-12, NAV-04, NAV-11:

    Extract search query from $args->{search} (LMS provides this for type => 'search' items). If empty, return NO_RESULTS textarea.

    For the initial search request (no pagination into sub-categories yet): call Client->search with q => $query, type => 'track,album,artist,playlist', limit => 10 (Dev Mode max per D-10 discretion).

    Build items array:
    1. Top Result (D-10): Take first track from $data->{tracks}{items}[0]. If exists, create an inline item (not a sub-menu) via _trackItem. Wrap in a section: { name => cstring TOP_RESULT, items => [_trackItem(...)], type => 'outline' }.
    2. For each type with results (tracks, albums, artists, playlists):
       - If the type's items array has entries, create a sub-menu item:
         { name => cstring($client, 'PLUGIN_SPOTON_TRACKS'), url => \&_searchTypeFeed, passthrough => [{ query => $query, type => 'track' }], type => 'link', line2 => "$total results" }
       - Categories with 0 results: skip entirely (D-10).

    Return $callback->({ items => \@items }).

    **B. _searchTypeFeed($client, $callback, $args, $passthrough)** — paginated drill-down into one search type (NAV-11):

    Extract query and type from $passthrough->[0]. Map LMS $args->{index} to offset, $args->{quantity} to limit (cap at 10 for Dev Mode). Call Client->search with q, type (single type string), limit, offset.

    The response key name varies: for type 'track' -> $data->{tracks}, 'album' -> $data->{albums}, etc. Map the plural form: track->tracks, album->albums, artist->artists, playlist->playlists.

    Map items through the appropriate builder:
    - tracks: _trackItem($client, $_)
    - albums: _albumItem($client, $_)
    - artists: _artistItem($client, $_)
    - playlists: _playlistItem($client, $_)

    Return with total => $type_data->{total} for LMS pagination.

    **C. _artistFeed($client, $callback, $args, $passthrough)** — per D-09, NAV-05:

    Extract artistId from $passthrough->[0]{artistId}. Build four section items, each pointing to a paginated sub-feed:
    - Albums: url => \&_artistAlbumsFeed, passthrough => [{ artistId => $id, includeGroups => 'album' }], name => cstring ALBUMS
    - Singles: passthrough includeGroups => 'single', name => cstring SINGLES
    - Compilations: passthrough includeGroups => 'compilation', name => cstring COMPILATIONS
    - Appears On: passthrough includeGroups => 'appears_on', name => cstring APPEARS_ON

    All four items type => 'link'. No Top Tracks section (NAV-10, D-09: removed in Dev Mode).
    No Related Artists section (NAV-10: removed in Dev Mode).

    Return $callback->({ items => \@items }).

    **D. _artistAlbumsFeed($client, $callback, $args, $passthrough)** — paginated per D-09 (separate request per group):

    Extract artistId and includeGroups from passthrough. Map LMS pagination to offset/limit (cap 50). Call Client->getArtistAlbums with include_groups => $includeGroups (SINGLE value, not combined — Pitfall 1). Map $data->{items} through _albumItem. Return with total.

    **E. _albumFeed($client, $callback, $args, $passthrough)** — per NAV-06:

    Extract albumId from passthrough. Two-step: first fetch album metadata via Client->getAlbum, then use the embedded tracks. If $args->{index} > 0 (user paginated past first page), use Client->getAlbumTracks with offset/limit instead.

    For initial load ($args->{index} == 0 or undef): use $album->{tracks}{items} (first page comes with album).
    For subsequent pages: use Client->getAlbumTracks.

    Build track items using a slightly modified format for album context (NAV-06: track number + duration + featuring artists):
    - line1: "$track_number. $title" (track number prefix)
    - line2: $artist (featuring artists — only show if different from album artist)
    - duration, play, url, on_select, image: from album images (not per-track since album tracks don't have images)
    - type: 'audio'

    For album tracks, the track object may NOT have album.images (it's a simplified track). Use the album's images from the initial getAlbum call. Pass album images via passthrough: passthrough => [{ albumId => $id, albumImages => $album->{images}, albumArtist => $album->{artists}[0]{name} }] when navigating to paginated view.

    Return with total => $data->{tracks}{total} (or $data->{total} for getAlbumTracks).

    **F. _playlistFeed($client, $callback, $args, $passthrough)** — per NAV-07:

    Extract playlistId from passthrough. Map LMS pagination to offset/limit (cap 100 for playlist items). Call Client->getPlaylistItems.

    Map $data->{items} through _trackItem (access $_->{track}). Skip null tracks (can happen with local files in playlists: check defined $_->{track}).

    Return with total => $data->{total}.

    **Made-For-You 403 fallback (RESEARCH Open Question 2):** Spotify-owned playlists (Made-For-You) may return 403 on getPlaylistItems if the Feb 2026 restriction blocks access. The error callback (undef $data) already returns NO_RESULTS textarea — this is the correct fallback. The Made-For-You feed in Home shows the playlist LIST (from getUserPlaylists); only drill-down into individual playlist tracks may fail. Accept this gracefully.

    **G. Add context navigation to _trackItem** (D-07):

    Modify the existing _trackItem function (from Plan 02) to include context menu items. Use the 'items' field on the track item itself. When an OPML item has both 'play' and 'items', LMS treats sub-items as context actions. Set:
    - items => [ { name => cstring($client, 'PLUGIN_SPOTON_ARTIST_VIEW'), url => \&_artistFeed, passthrough => [{ artistId => $track->{artists}[0]{id} }], type => 'link' }, { name => cstring($client, 'PLUGIN_SPOTON_ALBUM_VIEW'), url => \&_albumFeed, passthrough => [{ albumId => $track->{album}{id} }], type => 'link' } ]

    Only add artist link if $track->{artists}[0]{id} is defined. Only add album link if $track->{album}{id} is defined.

    **H. Add two i18n strings to strings.txt** (before the SON entry):
    - PLUGIN_SPOTON_ARTIST_VIEW — DE: "Kuenstler anzeigen" (use proper umlaut: the u-umlaut character), EN: "View Artist"
    - PLUGIN_SPOTON_ALBUM_VIEW — DE: "Album anzeigen", EN: "View Album"

    **Error handling:** All feed callbacks check for undef $data and return NO_RESULTS textarea.
