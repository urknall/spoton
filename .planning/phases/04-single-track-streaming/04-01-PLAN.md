<objective>
Implement the core transcoding engine that enables single-track Spotify streaming through LMS.

Purpose: This is the foundation for all audio playback — formatOverride dynamically selects the output format (OGG/FLAC) per player capabilities, and updateTranscodingTable injects runtime parameters (bitrate, cache dir, normalization, helper name) into the LMS commandTable before each track starts.

Output: Working ProtocolHandler.pm with dynamic format selection, Plugin.pm with updateTranscodingTable method and normalization pref initialization. After this plan, LMS can route spotify:// URIs through the correct transcoding pipeline per player.

Note: STR-09 (Gapless Playback) is listed as a Phase 4 ROADMAP requirement but is not deliverable with --single-track mode (separate librespot process per track). Per D-11 (CONTEXT.md), gapless is "nice-to-have" and a short pause between tracks is acceptable. No gapless implementation in this phase.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin.pm -> LMS TranscodingHelper | Plugin writes into global commandTable hashref; malformed values could break transcoding |
| Plugin.pm -> OS filesystem | Cache directory creation via make_path; path from server prefs (trusted) |
| ProtocolHandler.pm -> Helper.pm | getCapability reads from binary --check output (parsed JSON) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | updateTranscodingTable bitrate regex | mitigate | Bitrate value comes from prefs with whitelist validation in Settings.pm (only 96/160/320 accepted); regex `\d{2,3}` is safe |
| T-04-02 | Tampering | updateTranscodingTable helper name regex | mitigate | Helper name from Helper.pm::_findBin which uses Slim::Utils::Misc::findbin (trusted search paths only); basename() strips path components |
| T-04-03 | Tampering | updateTranscodingTable cacheDir in commandTable | mitigate | cacheDir from `preferences('server')->get('cachedir')` (LMS server config, not user input) + fixed suffix 'spoton'; no user-controlled path component |
| T-04-04 | Information Disclosure | commandTable logging at INFOLOG | accept | Only logged at INFOLOG level which requires explicit debug enable; shows command line with cache path (non-sensitive) |
</threat_model>
