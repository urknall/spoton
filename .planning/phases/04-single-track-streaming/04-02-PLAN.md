    2. ADD `playall => 1` to BOTH track item builders:

       a. In `_trackItem` (the `%item` hash, currently around line 264): add `playall => 1,` after `on_select => 'play',`. This enables context queueing for all track items across Search results, Recently Played, Top Tracks, Liked Songs, and Playlist tracks (D-09 and D-10).

       The playall flag tells XMLBrowser.pm to collect ALL items with playall=1 from the current feed and issue `playlist loadtracks listref \@urls undef $playIndex` — starting playback at the tapped track while queueing the entire feed. Per RESEARCH.md: this works automatically for already-loaded items in paginated feeds (limitation: only the current page is queued for large playlists).

    3. ADD ORPHANED PROCESS CLEANUP:

       a. In `initPlugin`, inside the `if ( !main::SCANNER )` block (currently lines 53-59), AFTER the existing token refresh timer setup, add:
          - `Slim::Utils::Timers::killTimers($class, \&_killOrphanedProcesses);`
          - `Slim::Utils::Timers::setTimer($class, Time::HiRes::time() + KILL_PROCESS_INTERVAL, \&_killOrphanedProcesses);`

       b. Add new sub `_killOrphanedProcesses`:
          - Receives `($class)` as argument (from Slim::Utils::Timers)
          - First: `Slim::Utils::Timers::killTimers($class, \&_killOrphanedProcesses)` to prevent duplicates
          - Check if any player is currently playing: iterate `Slim::Player::Client::clients()`, check `$client->isPlaying()`. If any is playing, set `$isBusy = 1` and skip killing.
          - If NOT busy: get helper via `Plugins::SpotOn::Helper->get()`. If found:
            - On Windows (`main::ISWINDOWS`): `system("taskkill /IM " . basename($helper) . " /F 1>nul 2>&1")`
            - On Unix: backtick `pkill -f "$helper"` (wrap in eval for safety)
          - Add comment `# PHASE-5-NOTE: Phase 5 must exclude Connect daemon PIDs here (Pitfall 6, CON-09)`
          - Always reschedule: `Slim::Utils::Timers::setTimer($class, Time::HiRes::time() + KILL_PROCESS_INTERVAL, \&_killOrphanedProcesses)`

    IMPORTANT: `Slim::Player::Client` does NOT need a `use` import — LMS autoloads it. Do NOT add `use Slim::Player::Client;`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings.pm POST handler | User-submitted form data (normalization checkbox value) |
| Plugin.pm -> OS process management | pkill command execution to kill orphaned processes |
| basic.html -> browser | Template rendering with user-visible pref values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | Settings.pm normalization pref | mitigate | Ternary normalization to 0 or 1 (`$paramRef->{'pref_normalization'} ? 1 : 0`); no arbitrary string accepted |
| T-04-06 | Tampering | _killOrphanedProcesses pkill | mitigate | `$helper` path comes from Helper.pm::_findBin (trusted search paths); never user-supplied. `pkill -f` matches only the binary path, not arbitrary patterns. Wrapped in eval for error containment. |
| T-04-07 | Denial of Service | _killOrphanedProcesses kills active processes | mitigate | isPlaying() check on all clients before killing; timer only fires when no player is active |
| T-04-08 | Tampering | playall flag manipulation | accept | playall is set server-side in Plugin.pm; not user-controllable input. XMLBrowser reads it from the feed array. |
</threat_model>
