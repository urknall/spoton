## Decisions Made

- `formatOverride` returns `'son'` (the INPUT content type) not `'flc'`/`'ogg'` (the output format). LMS `Song.pm` constructs the transcoding profile key as `formatOverride_return-outputFormat-*-*`. The prior code returning `'flc'` created `flc-flc-*-*` which had no matching entry in `custom-convert.conf`. `'son'` correctly matches `son-flc-*-*` and `son-ogg-*-*`.

- Passthrough-guard moved from `formatOverride` into `updateTranscodingTable`. Plan originally showed it in `formatOverride`, but centralizing all `commandTable` mutations in `updateTranscodingTable` keeps the logic in one place and avoids the need to re-import `CapabilitiesHelper` in `ProtocolHandler.pm`. The `CapabilitiesHelper` import was removed from `ProtocolHandler.pm` entirely.
