# Phase 04.1 Plan 02: librespot-spoton Single-Track Mode Summary

**One-liner:** librespot-playback 0.8 integrated into spoton binary with --single-track pipe-backend streaming, OGG-Vorbis passthrough, and accurate --check capability reporting.

## What Was Built

### Cargo.toml Changes
- Added `librespot-playback = { version = "0.8", default-features = false, features = ["passthrough-decoder", "rustls-tls-native-roots"] }`
- Added `[features]` section with `passthrough-decoder = []` and `default = ["passthrough-decoder"]` for accurate `--check` capability reporting

### main.rs Changes
- Added `SingleTrack` mode to `Mode` enum
- Added CLI argument parsing for: `--single-track`, `--bitrate`, `--passthrough`, `--start-position`, `--disable-discovery`, `--disable-audio-cache`, `--enable-volume-normalisation`
- Implemented `run_single_track()` async function:
  - Loads cached credentials from `cache_dir`
  - Creates Session and connects to Spotify
  - Configures `PlayerConfig` with bitrate, passthrough, normalisation flags
  - Uses `audio_backend::find(Some("pipe"))` for stdout output
  - Creates `Player::new` with `NoOpVolume` volume getter
  - Normalizes URI: `spotify://track:ID` → `spotify:track:ID`
  - Parses `SpotifyUri::from_uri()` for URI validation (T-04.1-05)
  - Starts playback with `player.load(track_id, true, start_position_ms)`
  - Waits for track end via `player.await_end_of_track()`
- Fixed `--check` to report `passthrough:true` and `ogg-direct:true` accurately via `cfg!(feature = "passthrough-decoder")`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Binary feature flag for --check passthrough reporting**
- **Found during:** Task 2
- **Issue:** `cfg!(feature = "passthrough-decoder")` warned "unexpected cfg condition value" because the feature belongs to librespot-playback, not the spoton binary itself. The plan suggested this approach but it would always evaluate to `false` without a mirrored feature.
- **Fix:** Added `[features]` section to binary Cargo.toml with `passthrough-decoder = []` and `default = ["passthrough-decoder"]`, mirroring the librespot-playback feature. This makes the binary's `--check` report accurately.
- **Files modified:** librespot-spoton/Cargo.toml
- **Commit:** 651f583

**2. [Rule 1 - Bug] Removed unused PlayerEvent import**
- **Found during:** Task 2 compilation
- **Issue:** Plan instructed importing `PlayerEvent` but `await_end_of_track()` was used instead of a manual event loop, making the import unused.
- **Fix:** Removed `PlayerEvent` from imports. Using `Player::await_end_of_track()` is cleaner and handles both `EndOfTrack` and `Stopped` events correctly.
- **Files modified:** librespot-spoton/src/main.rs
- **Commit:** 651f583

**3. [Rule 3 - Blocker] cargo check / cargo build without --target fails due to crt-static**
- **Found during:** Task 1 verification
- **Issue:** `.cargo/config.toml` sets `rustflags = ["-C", "target-feature=+crt-static"]` for `x86_64-unknown-linux-gnu`. This causes `proc-macro` crates (async-trait, used by librespot-playback) to fail compilation when invoked without explicit `--target` flag.
- **Impact:** The plan's verification step `cargo check` fails without `--target`. All build commands require `--target x86_64-unknown-linux-gnu` to bypass the implicit flag application.
- **Fix:** Used `--target x86_64-unknown-linux-gnu` in all build/check invocations. This is the established pattern in the project (see `.cargo/config.toml` comment).
- **Files modified:** None (operational knowledge only)
