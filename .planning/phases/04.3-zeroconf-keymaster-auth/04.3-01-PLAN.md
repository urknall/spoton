<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Spotify App -> librespot-discovery | ZeroConf HTTP server receives credentials from Spotify's proprietary protocol. librespot-discovery handles validation. |
| Binary stdout -> Perl layer | Username string from credentials.json passed to stdout. Perl consumer must validate format. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.3-01 | Spoofing | ZeroConf mDNS | accept | librespot-discovery uses Spotify's proprietary DiffieHellman+HMAC auth handshake — spoofing requires Spotify AP access. Risk: local network only, same as Spotty. |
| T-04.3-02 | Tampering | credentials.json write | mitigate | Cache dir created with default umask; Perl layer (Plan 02) will enforce chmod 700 on parent directory |
| T-04.3-03 | Info Disclosure | username on stdout | accept | stdout is consumed by Perl Proc::Background; not logged to external systems. Username is non-secret (visible in Spotify profile). |
| T-04.3-04 | Denial of Service | Discovery port binding | accept | Single port bound during discovery only (15-min max); released on exit. Same as Spotty. |
| T-04.3-SC | Tampering | cargo dependency install | mitigate | librespot-discovery is part of official librespot monorepo (same org as librespot-core already in use); futures-util is rust-lang official. Both [ASSUMED] — verified via cargo search. |
</threat_model>
