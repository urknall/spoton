    NEW METHOD: stopDiscovery($class) — kills discovery process if alive, kills watchdog timer, cleans __DISCOVER__ if no credentials.json inside.

    NEW METHOD: autoStartDiscoveryIfNeeded($class) — checks if any account has credentials.json in its cache subdir (iterate getAccountIds, check -f catfile(cachedir, 'spoton', $id, 'credentials.json')). If no accounts have credentials, call startDiscovery. Per D-01.

    NEW METHOD: _setupAccountFromCredentials($class, $cb) — called after discovery completes (from Settings AJAX flow):
    1. Read credentials.json from __DISCOVER__ dir
    2. Parse JSON, extract username field
    3. Derive accountId = substr(md5_hex($username), 0, 8)
    4. Rename __DISCOVER__ dir to {accountId} dir (File::Copy::move, remove_tree existing target first)
    5. chmod 0700 on the final dir (RESEARCH security: credential storage)
    6. Fetch display name: call _fetchKeymasterToken first to get a token, then use SimpleAsyncHTTP GET /me with that token to get display_name
    7. Store in prefs: accounts->{accountId} = { displayName => $displayName }. Set activeAccount if none.
    8. $cb->($accountId)
    Note: _setupAccountFromCredentials needs SimpleAsyncHTTP for the single /me call. Add "require Slim::Networking::SimpleAsyncHTTP" inside this method only (not at module top level).

    NEW METHOD: _fetchDisplayName($class, $accountId, $spotifyUserId, $cb) — gets token via _fetchKeymasterToken, then GET /me for display_name. Fallback: use spotifyUserId directly if /me fails.
