## Decisions Made

- Keymaster token via `--get-token` binary spawn (deferred via `setTimer`) replaces PKCE HTTP refresh ‚Äî no `Slim::Networking::SimpleAsyncHTTP` for token acquisition
- `Proc::Background` required on-demand inside `startDiscovery`, not at module top level (avoids LMS load-time failure on systems without Proc::Background)
- Plugin.pm uses thin `_autoStartDiscovery` callback wrapper (consistent with `_refreshAllTokens` pattern)
- `autoStartDiscoveryIfNeeded` checks `credentials.json` file existence rather than prefs alone (prefs could have stale account IDs after binary reinstall)
- `updateTranscodingTable` falls back to base `spoton/` dir when no `activeAccount` set ‚Äî prevents breakage at first boot before any account is configured
- D-07 confirmed: no migration from PKCE tokens needed (SpotOn in development, no external users)
