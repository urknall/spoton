## What Was Built

### Settings.pm (Task 1)

- `prefs()` method: added `clientId` to the returned list for LMS pref lifecycle management
- `handler()` saveSettings block: reads `pref_clientId` from form POST, applies T-04.4-01 input sanitization (`s/[^a-zA-Z0-9]//g` + `substr($id, 0, 32)`), saves via `$prefs->set('clientId', $id)`
- `handler()` template data block: adds `customClientId` and `degradedMode` template variables
- New `_isDegradedMode()` helper: returns 1 when `$prefs->get('clientId')` is falsy (no custom Client-ID configured), 0 otherwise — analog to existing `_isDiscoveryRunning()`

### basic.html (Task 2)

- New `[% WRAPPER setting %]` block for Client-ID input, inserted between NORMALIZATION block (line 24) and ZEROCONF block (`[% IF NOT accounts.keys.size %]`)
- Conditional degraded-mode warning: `[% IF degradedMode %]` shows orange div with warning text + anchor to `https://developer.spotify.com/dashboard` (target=_blank)
- Text input: `pref_clientId` with `value="[% customClientId | html %]"` and placeholder from strings

### strings.txt (Task 2)

5 new string keys appended after `PLUGIN_SPOTON_CONNECT_HINT_ALT`, each with DE and EN translations:

| Key | DE | EN |
|-----|----|----|
| PLUGIN_SPOTON_CLIENT_ID_SETTINGS | Eigene Spotify App | Custom Spotify App |
| PLUGIN_SPOTON_CLIENT_ID_DESC | Optionale eigene Spotify Developer App-ID für höhere Rate-Limits | Optional custom Spotify Developer App ID for higher rate limits |
| PLUGIN_SPOTON_CLIENT_ID_PLACEHOLDER | Spotify Client-ID (32 Zeichen, alphanumerisch) | Spotify Client ID (32 characters, alphanumeric) |
| PLUGIN_SPOTON_DEGRADED_MODE_WARNING | Keine eigene Spotify App-ID konfiguriert. Für volle Funktionalität bitte eigene App registrieren: | No custom Spotify App ID configured. For full functionality, please register your own app: |
| PLUGIN_SPOTON_REGISTER_APP_LINK | Eigene App registrieren | Register your own app |
