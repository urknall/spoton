**Hint-Cache-Methoden** (neue private Methoden, Quelle: Spotty-NG `API.pm:93-98`):

```perl
sub _lookupBundledHint {
    my ($cleanPath) = @_;
    for my $rx (@KNOWN_DEPRECATED_FAMILIES) {
        if ($cleanPath =~ $rx) {
            return 'bundled' if $cache->get(BUNDLED_HINT_KEY_PREFIX . md5_hex("$rx"));
        }
    }
    return undef;
}

sub _rememberBundledHint {
    my ($cleanPath) = @_;
    for my $rx (@KNOWN_DEPRECATED_FAMILIES) {
        if ($cleanPath =~ $rx) {
            my $key = BUNDLED_HINT_KEY_PREFIX . md5_hex("$rx");
            $cache->set($key, 1, BUNDLED_HINT_TTL);
            main::INFOLOG && $log->info("Client: bundled hint cached for $cleanPath");
            return;
        }
    }
}
```

**Fallback-Retry-Pattern** (neue `_doFlavouredRequest`-Methode, Quelle: Spotty-NG `API.pm:1595-1703`):

```perl
sub _doFlavouredRequest {
    my ($class, $method, $cleanPath, $params, $userCb, $flavor, $isRetry, $isMeFamily) = @_;

    require Plugins::SpotOn::API::TokenManager;
    Plugins::SpotOn::API::TokenManager->getToken($params->{_accountId}, $flavor, sub {
        my $token = shift;
        unless ($token) {
            $userCb->(undef, { error => 'no_token', flavor => $flavor });
            return;
        }
        $params->{_flavor} = $flavor;  # für _onError
        # HTTP-Call (bestehender Code aus _request Zeilen 281-325, extrahiert)
        $class->_executeHttp($token, $method, $cleanPath, $params, sub {
            my ($result, $err) = @_;
            my $code = $err ? ($err->{code} // 0) : 0;

            # 403/410/404-deprecated: Bundled-Fallback wenn nicht me/* und nicht bereits Retry
            if (!$isRetry && $flavor eq 'own' && !$isMeFamily
                    && ($code == 403 || $code == 410
                        || ($code == 404 && _is404Deprecated($cleanPath)))) {
                $class->_doFlavouredRequest(
                    $method, $cleanPath, $params, $userCb, 'bundled', 1, $isMeFamily);
                return;
            }

            # Hint cachen wenn bundled-Retry erfolgreich (Hint nur bei 2xx schreiben)
            if (!$err && $isRetry && $flavor eq 'bundled') {
                _rememberBundledHint($cleanPath);
            }

            $userCb->($result, $err);
        });
    });
}
```

## Anti-Patterns (aus RESEARCH.md)

| Anti-Pattern | Fundstelle | Vermeidung |
|---|---|---|
| Double-Callback | Spotty-NG `API.pm:1588-1593` | `$userCbCalled`-Guard in `_request()` |
| Recursion im Fallback | Spotty-NG `API.pm:1380-1383` | `_doFlavouredRequest()` direkt aufrufen, nicht `_request()` erneut |
| Hint vor Erfolg schreiben | Spotty-NG `API.pm:1656-1662` | `_rememberBundledHint()` nur wenn `!$err && $isRetry && $flavor eq 'bundled'` |
| me/* auf bundled fallen lassen | Spotty-NG `API.pm:107-112` | me/*-Guard ist erste Routing-Entscheidung, niemals überspringen |
| 404 global als deprecated-Signal | RESEARCH Pitfall 4 | 404 → bundled-Retry nur wenn `_is404Deprecated($cleanPath)` true |
| Hint-Cache-Key aus URL-String | RESEARCH Pitfall 5 | `md5_hex("$matchedRx")` — Key aus Regex-Pattern, nicht URL |
| Flavor-spezifische Concurrency-Caps | RESEARCH Pitfall 6 | `MAX_CONCURRENT_REQUESTS = 3` gilt shared für beide Flavors |
