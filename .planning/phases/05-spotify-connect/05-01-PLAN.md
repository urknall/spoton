<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binary HTTP server | localhost-only but accepts TCP connections from any process on the host |
| JSON-RPC to LMS | Binary POSTs to LMS /jsonrpc.js — treated as trusted (same host) |
| /control/* endpoints | Accept JSON body from localhost — must validate input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | LMS::new() auth header | mitigate | trim() + replace(\r\n) on auth string in constructor (CRLF injection prevention) |
| T-05-02 | Tampering | /control/volume JSON body | mitigate | Parse volume as u32, clamp to 0-100 range before conversion to u16 |
| T-05-03 | Tampering | /control/seek JSON body | mitigate | Parse position_ms as u64, reject negative values |
| T-05-04 | DoS | /stream concurrent connections | mitigate | relay_active AtomicBool prevents split-PCM-stream (CR-01) |
| T-05-05 | DoS | HttpStreamSink::write exit() | mitigate | No exit() in stop() — daemon survives track boundaries (Pitfall 1) |
| T-05-06 | Spoofing | spirc_active race | mitigate | Set true ONLY on SessionConnected; HTTP server returns 503 until ready (Pitfall 2) |
| T-05-SC | Tampering | cargo install | mitigate | All packages verified in RESEARCH.md Package Legitimacy Audit — established Rust ecosystem packages |
</threat_model>
