    Create Plugins/SpotOn/Connect.pm adapting Spotty-NG Connect.pm.

    **Package:** Plugins::SpotOn::Connect

    **Constants:** SEEK_THRESHOLD = 3, VOLUME_GRACE_PERIOD = 20 (CON-11), CONNECT_START_GRACE = 12, VOLUME_DEBOUNCE = 0.5, SEEK_DEBOUNCE = 0.3.

    **Imports:** strict, File::Path mkpath, File::Spec::Functions catdir catfile, JSON::XS::VersionOneAndTwo, Scalar::Util blessed, Time::HiRes, Slim::Utils::Log, Slim::Utils::Cache, Slim::Utils::Prefs, Slim::Utils::Timers, Slim::Networking::SimpleAsyncHTTP (for HTTP control endpoint calls).

    **State:** my $initialized, my $_activeConnectPlayer, $prefs, $serverPrefs, $log.

    **init($class):** Return if $initialized. Register spottyconnect dispatch via addDispatch(['spottyconnect', '_cmd'], [1, 0, 1, \&_connectEvent]). Subscribe to 5 event types: playlist/newsong, playlist/pause+stop, mixer/volume, time (seek), playlist/jump+index. Require and init DaemonManager. Set $initialized = 1.

    **isSpotifyConnect($class, $client):** Return true if $_activeConnectPlayer equals $client->id (or master->id). Used by ProtocolHandler to disable seek during Connect.

    **_connectEvent($request):** Main event handler from binary JSON-RPC. Get $client = $request->client()->master, $cmd = getParam('_cmd'). Dispatch based on $cmd:
    - 'start': Set pendingConnect pluginData, set $_activeConnectPlayer. Stop any active Browse playback on this player (D-08 mutual exclusion). Get track_id from _p2 param. Store progress in pluginData BEFORE playlist play (CON-17). Issue $client->execute(['playlist', 'play', "spotify://connect-$ts"]) with source(__PACKAGE__).
    - 'change': Get new_track_id from _p2 and old_track_id from _p3. Fetch metadata via API::Client->getTrack(). Use event trackId over API response if they disagree (stale-API protection per Pitfall 5). Update metadata.
    - 'stop': Execute ['pause', 1] with source marking.
    - 'volume': Check grace period via DaemonManager->uptime (CON-11). If uptime < VOLUME_GRACE_PERIOD, return. Otherwise set mixer volume via Slim::Control::Request with source(__PACKAGE__) to prevent loop (Pitfall 6).
    - 'seek': Use startOffset pattern (CON-13). If pendingConnect, store as initial progress. Otherwise $song->startOffset(int($position) - $client->songElapsedSeconds()). NEVER use ['time', N] in stream mode (Pitfall 7).

    **_onPause($request):** Return early if source is __PACKAGE__. Check if client has active Connect. POST to http://127.0.0.1:$port/control/pause via SimpleAsyncHTTP (D-14).

    **_onVolume($request):** Return early if source is __PACKAGE__. Check if client has active Connect. Debounce 0.5s via killTimers + setTimer. POST to /control/volume with JSON body {"volume": $volume} (D-14).

    **_onSeek($request):** Return early if source is __PACKAGE__. Check if client has active Connect. Debounce 0.3s via killTimers + setTimer. POST to /control/seek with JSON body {"position_ms": $positionMs} (D-14).

    **_onPlaylistJump($request):** Return early if source is __PACKAGE__. POST to /control/next or /control/prev based on direction.

    **_onNewSong($request):** Handle CON-17 progress-before-play for the new song. Check if this is a Connect session via URL pattern (spotify://connect-*).

    **_sendControlCommand($client, $endpoint, $body):** Helper for HTTP control POSTs. Build URL from DaemonManager->streamPortForClient. Use SimpleAsyncHTTP with 5s timeout. On error, fall back to Spotify Web API via API::Client (D-15).

    **_stopConnectDaemon($client):** Stop the Connect daemon for this client, clear $_activeConnectPlayer. Called when Browse starts (D-08 mutual exclusion).

    Source-marking is CRITICAL: every outbound Slim::Control::Request MUST use $req->source(__PACKAGE__). Every subscriber MUST check $request->source eq __PACKAGE__ and return early.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binary JSON-RPC events | Received from localhost binary via spottyconnect dispatch — treated as trusted |
| HTTP control endpoints | Plugin POSTs to localhost binary — no external attack surface |
| canDirectStream URL | Constructed from localhost + daemon port — no user input in URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-13 | Repudiation | Volume/seek feedback loop | mitigate | source(__PACKAGE__) on ALL outbound commands + early return in ALL subscribers |
| T-05-14 | Tampering | Stale API response on change | mitigate | Binary event p2 (new_track_id) has priority over API response (Pitfall 5) |
| T-05-15 | DoS | ['time', N] restart in stream mode | mitigate | startOffset ONLY in _connectEvent seek handler (Pitfall 7, CON-13) |
| T-05-16 | Spoofing | canDirectStream URL injection | accept | URL constructed from Slim::Utils::Network::serverAddr() + daemon port — both LMS-controlled, no user input |
| T-05-17 | DoS | Mutual exclusion failure | mitigate | _connectEvent 'start' stops Browse playback; Plugin.pm Browse start stops Connect daemon (D-08) |
| T-05-SC | Tampering | Module requires | mitigate | All modules project-local or LMS-bundled |
</threat_model>
