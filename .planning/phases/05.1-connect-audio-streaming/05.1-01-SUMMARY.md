## What Was Built

### Task 1: Grace-Timer + Debug Logging (connect.rs)

Three changes to `librespot-spoton/src/connect.rs`:

**Change A — LMS struct: `last_session_start_ns: Arc<AtomicU64>` field**
- Added to LMS struct after `needs_position_sync`
- Initialized in `LMS::new()` as `Arc::new(AtomicU64::new(0))`
- Cloned in `Clone` impl as `Arc::clone(&self.last_session_start_ns)`
- Follows exact same pattern as existing `seek_gen: Arc<AtomicU64>`

**Change B — Grace-timer logic in `handle_player_event` (D-03)**
- `TrackChanged` arm, `None` branch (session start): stores `SystemTime::now()` as nanoseconds-since-UNIX-EPOCH in `last_session_start_ns` via `Ordering::Release`
- Grace timer set ONLY on `None->Some` transition (first TrackChanged of a session)
- `Paused/Stopped` arm: checks grace timer before calling `notify("stop")`. If `last > 0` and elapsed < 2s, logs suppression message and returns early without clearing `current_track`
- User-initiated Pause during track-to-track transitions is never suppressed (grace timer only set at session start)

**Change C — eprintln! debug logging (D-02)**
- `Playing` arm: `[spoton] Playing: track_id={new_id}, position_ms={position_ms}`
- `Paused/Stopped` arm: `[spoton] Paused/Stopped (disc={:?}): current_track={:?}` (discriminant distinguishes Paused vs Stopped)
- `VolumeChanged` arm: `[spoton] VolumeChanged: volume={volume}`
- `Seeked` arm: `[spoton] Seeked: position_ms={position_ms}`
- `SessionConnected` arm: `[spoton] SessionConnected`
- `TrackChanged` arm: `[spoton] TrackChanged: track_id={new_id}`
- `TrackChanged` None branch: `[spoton] TrackChanged (session start): grace timer set`
- Grace suppression: `[spoton] Paused/Stopped suppressed (grace timer, {ms}ms elapsed)`
- `HttpStreamSink::write()`: `[spoton] sink write: {frames_consumed} frames consumed` every 1000 frames
