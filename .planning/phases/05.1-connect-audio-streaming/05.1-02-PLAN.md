    Four changes to Daemon.pm:

    **Change A — Import catfile**

    Extend the File::Spec::Functions import on line 8 from `qw(catdir)` to `qw(catdir catfile)`.

    **Change B — Add _stderrFh accessor**

    Add `_stderrFh` to the mk_accessor list (after _streamPort on line 33). This prevents Perl from garbage-collecting the file handle while the binary process is still running (RESEARCH.md Pitfall 3).

    **Change C — Open stderr log file before Proc::Background spawn**

    Insert between the `pipe()` call (line 117-118) and the `eval { ... Proc::Background->new` block (line 120):

    1. Construct the log file path using catfile: `catfile($serverPrefs->get('cachedir'), 'spoton', $self->id . '-connect.log')`. This places it next to the cache directory, using the sanitized MAC-based id (colons already stripped in new()).
    2. Open the file handle in append mode (`>>`). If open fails, log a warning and set $stderr_fh to undef (non-fatal — binary runs without stderr capture).
    3. Store the file handle: `$self->_stderrFh($stderr_fh)` after the spawn succeeds.

    **Change D — Pass stderr to Proc::Background**

    Modify the Proc::Background->new() options hash on line 122. Add `stderr => $stderr_fh` conditionally: only if $stderr_fh is defined. Use the ternary pattern from PATTERNS.md:
    `{ 'die_upon_destroy' => 1, stdout => $port_w, ($stderr_fh ? (stderr => $stderr_fh) : ()) }`

    CRITICAL ordering: close($port_w) happens AFTER spawn (line 129) — this is already correct. Do NOT close $stderr_fh — it must remain open for the lifetime of the process. The _stderrFh accessor holds the reference.

    Also add an INFOLOG line after successful start (near line 156-158) that logs the stderr log file path: `"SpotOn Connect daemon stderr logged to $stderrFile"`.
