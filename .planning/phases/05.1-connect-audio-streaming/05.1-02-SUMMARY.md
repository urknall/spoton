## Accomplishments

- Added `catfile` to File::Spec::Functions import (alongside existing `catdir`)
- Added `_stderrFh` accessor to `mk_accessor` list to prevent premature GC of log file handle
- Opened log file in append mode (`>>`) before Proc::Background spawn, with non-fatal fallback

- Stored handle as `_stderrFh` accessor after successful spawn

## Decisions Made

- Non-fatal open failure pattern: `undef $stderrFile; undef $stderr_fh` — binary continues without capture if log directory does not exist yet
- `_stderrFh` assigned after spawn success (inside success path, not before) so the GC-guard only activates when the process actually started
- `$stderr_fh` is intentionally never closed in `start()` — Proc::Background needs the FD open for the child's lifetime; only object destruction (via `_stderrFh` going out of scope) closes it
