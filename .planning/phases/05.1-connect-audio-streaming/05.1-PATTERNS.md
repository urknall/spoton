### `librespot-spoton/src/connect.rs` — Bug 1: Grace-Timer in handle_player_event

**Fix-Ziel:** Spirc feuert `PlayerEvent::Paused` (Discriminant 5) kurz nach
`PlayerEvent::TrackChanged` (Discriminant 13). Das Binary sendet daraufhin ein
spurious "stop" an LMS, das die Connect-Session abbricht.

**Einfuegepunkt:** Funktion `handle_player_event`, Zeilen 133–233.

**Bestehender Paused/Stopped-Arm** (Zeilen 174–177):
```rust
// Zeile 174-177 — dieser Arm loest Bug 1 aus
PlayerEvent::Paused { .. } | PlayerEvent::Stopped { .. } => {
    if current_track.take().is_some() {
        self.notify("stop", "", "").await;
    }
}
```

**Bestehender TrackChanged-Arm, None-Branch** (Zeilen 222–226):
```rust
// Zeile 222-226 — hier wird der Grace-Timer gesetzt
None => {
    self.needs_position_sync.store(true, Ordering::Release);
    *current_track = Some(new_id.clone());
    self.notify("start", &new_id, "").await;
}
```

**Pattern fuer den Grace-Timer (aus RESEARCH.md Pattern 1):**

Bestehende Arc-Atomic-Infrastruktur im LMS-Struct als Muster:
```rust
// Zeile 21 — vorhandene Imports, ergaenzen mit std::time::Instant
use std::sync::{Arc, atomic::{AtomicBool, AtomicU64, Ordering}};
use std::time::{Duration, Instant};

// Zeile 70-76 — LMS-Struct, neues Feld analog zu seek_gen: Arc<AtomicU64>
pub struct LMS {
    pub host_port: Option<String>,
    pub player_mac: Option<String>,
    pub auth: Option<String>,
    pub suppress_next_volume: Arc<AtomicBool>,
    pub flush_tx: Option<watch::Sender<u64>>,
    pub seek_gen: Arc<AtomicU64>,
    pub needs_position_sync: Arc<AtomicBool>,
    // NEU: Timestamp (Nanos seit UNIX-Epoch) des letzten None->Some TrackChanged
    pub last_session_start_ns: Arc<AtomicU64>,  // <-- hinzufuegen
}
```

**Implementierungsmuster** — analog zur bestehenden suppress_next_volume-Logik
(Zeilen 184–189):
```rust
// Im TrackChanged-Arm, None-Branch: Timestamp setzen
None => {
    // Grace-Timer: nur bei Session-Start (None->Some), nicht bei Track-Wechseln
    let now_ns = std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .unwrap_or_default()
        .as_nanos() as u64;
    self.last_session_start_ns.store(now_ns, Ordering::Release);

    self.needs_position_sync.store(true, Ordering::Release);
    *current_track = Some(new_id.clone());
    self.notify("start", &new_id, "").await;
}

// Im Paused/Stopped-Arm: Grace-Timer pruefen
PlayerEvent::Paused { .. } | PlayerEvent::Stopped { .. } => {
    let grace_ns = 2_000_000_000u64; // 2 Sekunden
    let last = self.last_session_start_ns.load(Ordering::Acquire);
    if last > 0 {
        let now_ns = std::time::SystemTime::now()
            .duration_since(std::time::UNIX_EPOCH)
            .unwrap_or_default()
            .as_nanos() as u64;
        if now_ns.saturating_sub(last) < grace_ns {
            eprintln!("[spoton] Paused/Stopped suppressed (grace timer active, {}ms elapsed)",
                now_ns.saturating_sub(last) / 1_000_000);
            return;
        }
    }
    if current_track.take().is_some() {
        self.notify("stop", "", "").await;
    }
}
```

**Clone-Impl — Muster analog zu seek_gen** (Zeilen 105–119):

**new()-Initialisierung — Muster analog zu seek_gen** (Zeilen 79–96):

### `librespot-spoton/src/connect.rs` — Bug 2: HTTP-Response-Headers fuer squeezelite

**Fix-Ziel:** squeezelite sendet `Range: bytes=0-` (via LMS HTTP.pm::requestString).
Binary antwortet mit `200 OK`. Falls squeezelite 206 erwartet, schlaegt die Verbindung fehl.

**Einfuegepunkt:** `/stream` Response-Builder, Zeilen 596–601.

**Bestehender Response-Builder** (Zeilen 596–601):
```rust
// Zeile 596-601 — bestehender /stream-Response
let resp = Response::builder()
    .status(StatusCode::OK)
    .header("Content-Type", "audio/L16;rate=44100;channels=2")
    .body(body)
    .expect("static 200 stream response builder");
```

**Optionale Ergaenzung bei Range-Request (falls squeezelite 206 benoetigt):**
```rust
// Pruefe ob Range-Header vorhanden — RFC 2616: Server darf Range ignorieren und 200 antworten.
// squeezelite 1.9.9 akzeptiert 200 OK bei Range-Request (per RESEARCH.md Open Question 1).
// Falls nicht: diesen Block aktivieren:
let has_range = req.headers().contains_key("range");
let (status, extra_headers) = if has_range {
    (StatusCode::PARTIAL_CONTENT, Some(("Content-Range", "bytes 0-*/*")))
} else {
    (StatusCode::OK, None)
};
let mut builder = Response::builder()
    .status(status)
    .header("Content-Type", "audio/L16;rate=44100;channels=2");
if let Some((k, v)) = extra_headers {
    builder = builder.header(k, v);
}
let resp = builder.body(body).expect("stream response builder");
```

**Prioritaet:** Erst squeezelite `-d stream:debug` Log auswerten (D-01), dann
entscheiden ob Range-Fix noetig ist.

## Shared Patterns

### Atomic-State-Sharing zwischen Tasks

**Quelle:** `connect.rs` Zeilen 70–76, 105–119
**Anwenden auf:** Grace-Timer (`last_session_start_ns`)

```rust
// Muster: Arc<AtomicU64> — shared zwischen LMS-Notifier-Task und Clone
pub seek_gen: Arc<AtomicU64>,  // bestehend — Grace-Timer folgt exakt diesem Pattern
```

Der neue `last_session_start_ns: Arc<AtomicU64>` folgt identisch dem `seek_gen`-Pattern:
- In `LMS::new()`: `Arc::new(AtomicU64::new(0))`
- In `Clone`: `Arc::clone(&self.last_session_start_ns)`
- Zugriff: `store(val, Ordering::Release)` / `load(Ordering::Acquire)`

### Perl-Accessor-Pattern fuer Filehandles

**Quelle:** `Daemon.pm` Zeilen 22–34
**Anwenden auf:** `_stderrFh`

```perl
# Muster: mk_accessor rw => qw(...)
# Alle internen Felder werden so deklariert, inkl. komplexe Typen (_proc, _startTimes)
# _stderrFh folgt exakt diesem Pattern
__PACKAGE__->mk_accessor( rw => qw( ... _stderrFh ) );
```

### eprintln!-Logging-Stil

**Quelle:** `connect.rs` Zeile 966, 995
**Anwenden auf:** Alle neuen Log-Ausgaben in `handle_player_event` und `HttpStreamSink::write`
