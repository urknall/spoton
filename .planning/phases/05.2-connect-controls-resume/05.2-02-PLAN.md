## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binary -> Connect.pm | spottyconnect JSON-RPC dispatch — _p2/_p3 params from binary notify() |
| Connect.pm -> LMS | ['pause', 0] command — source-marked to prevent subscriber echo |
| Client.pm -> Spotify API | PUT requests with Content-Length: 0 — no user-controllable data in this fix |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.2-04 | Spoofing | resume _p3 position param | accept | Position value comes from binary (own process, localhost JSON-RPC). int() typecast in Perl prevents non-numeric exploitation. startOffset accepts negative values by design. |
| T-05.2-05 | Tampering | ['pause', 0] command injection | mitigate | Source-marking via `$req->source(__PACKAGE__)` prevents _onPause subscriber from echoing the command back to Spotify, preventing control loop amplification (T-05-13 pattern). |
| T-05.2-06 | Repudiation | Content-Length: 0 on all PUT/POST | accept | This is a standards-compliance fix, not a behavioral change. No audit logging impact. |
| T-05.2-SC | Tampering | No package installs in this plan | accept | Plan modifies only existing Perl files. No npm/pip/cargo installs. |
</threat_model>
