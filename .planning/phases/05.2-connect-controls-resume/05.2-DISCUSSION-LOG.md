## Resume-Mechanismus

### Frage 1: Wie soll das Binary LMS über Resume informieren?

| Option | Description | Selected |
|--------|-------------|----------|
| Neues 'resume' Kommando | Wire-Vokabular um 'resume' erweitern. Binary sendet notify('resume', track_id, position). Perl-Seite: _connectEvent bekommt neuen 'resume'-Handler der ['pause', 0] (unpause) ausführt. | ✓ |
| Start wiederverwenden | Kein neues Kommando. Playing im None-Branch sendet 'start'. Perl-Seite muss erkennen ob echter Start oder Resume. | |
| Paused-State beibehalten | current_track bei Paused/Stopped NICHT clearen (take() entfernen). Playing im Same-Track-Branch sendet dann notify('seek', position). | |
| Claude entscheidet | Researcher/Planner wählt basierend auf Spotty-NG-Referenz. | |

**User's choice:** Neues 'resume' Kommando
**Notes:** Sauber und explizit bevorzugt
