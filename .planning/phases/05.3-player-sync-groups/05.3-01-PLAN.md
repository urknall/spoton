<objective>
Fix the sync-group proxy disconnect bug by overriding two methods in ProtocolHandler.pm.

Purpose: When LMS proxies Connect audio for synced players, it calls ProtocolHandler::new() which
substitutes the spotify://connect-* URL with http://127.0.0.1:PORT/stream and delegates to
Slim::Player::Protocols::HTTP->new(). Two issues cause immediate disconnection:
(1) The base class requestString() unconditionally adds "Range: bytes=0-" which is nonsensical
for an infinite PCM stream and may trigger different hyper response handling.
(2) HTTP.pm::response() enters the "Persistent service" reconnect path when no Content-Length
is present (!$self->contentLength at line 107), even if canEnhanceHTTP returns 0 — the Persistent
path tries ranged reconnections against an infinite stream, causing disconnect.

Output: ProtocolHandler.pm with two new method overrides that eliminate both disconnect causes.
</objective>
