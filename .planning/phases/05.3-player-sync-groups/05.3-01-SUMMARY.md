# Phase 05.3 Plan 01: requestString + canEnhanceHTTP overrides for Connect proxy

**One-liner:** ProtocolHandler.pm gets two method overrides that suppress Range header and block LMS's Persistent reconnect path for Connect proxy connections to the binary's /stream endpoint.

## What Was Built

Added two method overrides to `Plugins/SpotOn/ProtocolHandler.pm`:

**`requestString` override:** When LMS calls this for a Connect proxy URL (`http://127.0.0.1:PORT/stream`), the base class (`Slim::Player::Protocols::HTTP`) unconditionally adds `Range: bytes=0-` (HTTP.pm line 971). This is wrong for an infinite PCM stream — the binary's hyper HTTP server may respond with 206 Partial Content and the Range header contributes to triggering LMS's Persistent reconnect path. The override intercepts URLs matching `m{:\d+/stream\b}`, builds a plain `GET /stream HTTP/1.0` request without any Range header, and delegates all other URLs to `SUPER::requestString`.

**`canEnhanceHTTP` override:** The base class returns `$prefs->get('useEnhancedHTTP')` which may be non-zero. When it's non-zero, LMS's `HTTP.pm::response()` at line 107 enters the "Persistent service" path (`readPersistentChunk`) for connections with no Content-Length. This path attempts range-based reconnections against an infinite stream, causing immediate disconnection. The override returns 0 for `m{:\d+/stream\b}` URLs, forcing LMS to use the normal streaming path. All other URLs delegate to `SUPER::canEnhanceHTTP`.

**`$CRLF` constant:** Added `my $CRLF = "\x0d\x0a"` at package level (the base class does not export it).
