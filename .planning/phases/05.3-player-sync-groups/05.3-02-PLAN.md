<objective>
Add relay_active safety clearing and Spirc reconnect 'ready' notification to the binary.

Purpose: Two binary-side improvements for sync-group robustness and session handover:
(1) When Spirc reconnects after session death, relay_active may be stuck true from a previous
relay task that didn't run its Drop guard (tokio task cancellation). Explicitly clearing
relay_active in the reconnect branch prevents permanent 503 responses on /stream.
(2) After successful Spirc reconnect, notify LMS with a 'ready' command so the Perl side
can re-issue playlist play and resume streaming without user intervention (D-04 session handover).

Output: Modified connect.rs with both fixes + rebuilt binary.
</objective>
