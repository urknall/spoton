## Decisions Made

- `connectStartTime` is set inside the `$_activeConnectPlayer` guard (not outside) — only reset the grace period if we actually re-issue play. This avoids spurious grace-period activation for inactive players.
- Log message is emitted before the guard check so all 'ready' events are visible in LMS log for debugging, even if the player is no longer the active Connect player.

## Deviations from Plan

None — plan executed exactly as written for Tasks 1 and 2.

## Issues Encountered

Three bugs found and fixed during UAT (commit 54a0963):
1. **Resume handler URL check**: `isSpotifyConnect()` checked `$_activeConnectPlayer` flag (stays set after source switch) instead of actual `streamUrl`. Fixed to check `$song->streamUrl =~ m{spotify://connect-}`.
2. **Missing newTrack suppression**: Resume re-enter path lacked `newTrack => 1` / `connectPauseTs => 0` / `connectStartTime`, causing transitional pause to be forwarded to binary → immediate re-pause. Fixed with same suppression flags as `start` handler.
3. **Binary 503 on stale relay**: `/stream` returned 503 when `relay_active` was true from idle relay (no PCM data, client disconnected but not detected). Changed from reject-503 to connection-takeover — new request force-clears and proceeds.
