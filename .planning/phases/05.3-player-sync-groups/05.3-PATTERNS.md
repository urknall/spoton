### `Plugins/SpotOn/ProtocolHandler.pm` — two new methods: `requestString` and `canEnhanceHTTP`

**Analog:** `/home/sti/spotty-ng/Spotty-Plugin/ProtocolHandler.pm` + `/usr/share/perl5/Slim/Player/Protocols/HTTP.pm`

**Problem being fixed:** When `new()` substitutes the `spotify://connect-*` URL with
`http://127.0.0.1:PORT/stream`, LMS calls `requestString()` on the resulting
`Slim::Player::Protocols::HTTP` object. That method adds `Range: bytes=0-` unconditionally
(HTTP.pm line 971). The binary's hyper HTTP server responds, but LMS's `response()` at
line 107 also checks `!$self->contentLength` — when no `Content-Length` header is present,
it unconditionally enters the "Persistent service" reconnect path even if `canEnhanceHTTP`
returns 0. The combination of a Range header the binary may handle differently, plus the
Persistent path trying ranged reconnections against an infinite stream, causes disconnection.

**Two fixes required:**

**Fix A — override `requestString` to suppress Range for Connect proxy URLs.**

Base class `requestString` from `/usr/share/perl5/Slim/Player/Protocols/HTTP.pm` (lines 921-984):
```perl
sub requestString {
    my $self   = shift;
    my $client = shift;
    my $url    = shift;
    my $post   = shift;
    my $seekdata = shift;

    my ($server, $port, $path, $user, $password, $proxied) = Slim::Utils::Misc::crackURL($url);
    $path = $proxied if $proxied;
    my $type = $post ? 'POST' : 'GET';
    my $host = (($url =~ /^http:/ && $port == 80) || ($url =~ /^https:/ && $port == 443))
        ? $server : "$server:$port";

    my $request = join($CRLF, (
        "$type $path HTTP/1.0",
        "Accept: */*",
        "Cache-Control: no-cache",
        "User-Agent: " . Slim::Utils::Misc::userAgentString('legacy'),
        "Icy-MetaData: $want_icy",
        "Connection: close",
        "Host: $host",
    ));

    # Adds Range: bytes=0- unconditionally (line 971) — skip for infinite streams.
    $request .= $CRLF . 'Range: bytes=' . ($first || 0) . '-';
    ...
}
```

SpotOn's override (to add to `ProtocolHandler.pm`):
```perl
# Override requestString to suppress Range header for Connect proxy connections.
# The base class unconditionally adds "Range: bytes=0-" (HTTP.pm line 971).
# Sending Range to an infinite PCM stream endpoint causes hyper to respond with
# 206 Partial Content, and more critically triggers LMS's "Persistent service"
# reconnect path (HTTP.pm line 107: !$self->contentLength with Range present).
# For the binary's /stream endpoint, a plain GET without Range is correct.
my $CRLF = "\x0d\x0a";

sub requestString {
    my ($self, $client, $url, $post, $seekdata) = @_;

    # Only intercept the Connect proxy URL pattern (http://127.0.0.1:PORT/stream).
    # All other URLs (Browse/single-track son:// or external http://) use the
    # base class behaviour which adds Range.
    if ($url && $url =~ m{:\d+/stream\b}) {
        my ($server, $port, $path) = Slim::Utils::Misc::crackURL($url);
        my $host = ($port == 80) ? $server : "$server:$port";
        return join($CRLF,
            "GET $path HTTP/1.0",
            "Accept: */*",
            "Cache-Control: no-cache",
            "Connection: close",
            "Host: $host",
            "", "",
        );
    }

    return $self->SUPER::requestString($client, $url, $post, $seekdata);
}
```

**Fix B — override `canEnhanceHTTP` to return 0 for Connect proxy URLs.**

Base class from `/usr/share/perl5/Slim/Player/Protocols/HTTP.pm` (line 413):
```perl
sub canEnhanceHTTP {
    return $prefs->get('useEnhancedHTTP');
}
```

`canEnhanceHTTP` is called with `($self, $client, $url)` at HTTP.pm line 104. If it returns
any true value, LMS sets up the `_enhanced` hash and calls `readPersistentChunk` — which
tries range-based reconnections against an infinite stream and immediately disconnects.
If it returns 0, LMS skips the Persistent path entirely. The override:

```perl
# canEnhanceHTTP: return 0 for Connect proxy stream URLs.
# The base class returns $prefs->get('useEnhancedHTTP') which may be non-zero.
# For infinite PCM streams the "Enhanced/Persistent" path causes immediate
# disconnection (HTTP.pm line 107 readPersistentChunk against an infinite body).
# Returning 0 forces LMS to use the normal non-enhanced streaming path.
sub canEnhanceHTTP {
    my ($self, $client, $url) = @_;
    return 0 if $url && $url =~ m{:\d+/stream\b};
    return $self->SUPER::canEnhanceHTTP($client, $url);
}
```

**Existing imports (lines 1-17 of ProtocolHandler.pm) — no new imports needed:**
```perl
use Slim::Utils::Network;     # already used for serverAddr()
use Slim::Utils::Misc;        # crackURL — already available via base class
```

Note: `$CRLF` must be defined as a package-level constant (the base class does not export it).
Check whether `Slim::Formats::RemoteStream` exports `$CRLF`; if not, define locally:
```perl
my $CRLF = "\x0d\x0a";
```

### `librespot-spoton/src/connect.rs` — two targeted fixes

**Analog:** The file itself. Two independent changes:

#### Fix 1 — relay_active safety: force-clear on reconnect

**Location context:** `run_connect()` Spirc reconnect branch (lines 1048-1074).

**Pattern for spirc_active reset** (existing code, lines 1056-1057):
```rust
// Spirc task completed unexpectedly — attempt reconnect with backoff
current_spirc_task = None;
spirc_active.store(false, Ordering::SeqCst);  // already clears spirc_active
```

**Gap to fix:** `relay_active` inside `http_stream_server` is a separate `Arc<AtomicBool>`.
The relay task's `RelayGuard::drop()` (lines 599-602) clears it on normal exit:
```rust
struct RelayGuard(Arc<AtomicBool>);
impl Drop for RelayGuard {
    fn drop(&mut self) { self.0.store(false, Ordering::Release); }
}
let _guard = RelayGuard(Arc::clone(&relay_active_clone));
```

**Pitfall 1 risk:** If the tokio relay task is *cancelled* (JoinHandle dropped without
`await`), `Drop` may not run on the task-local `_guard`. This leaves `relay_active=true`
permanently. All subsequent `/stream` requests return 503 (relay-busy guard at line 570).

**Fix:** Pass `relay_active` Arc into `run_connect()` alongside `spirc_active`, then in
the Spirc reconnect branch explicitly clear it:
```rust
// After spirc_task completes unexpectedly (reconnect branch, after line 1057):
spirc_active.store(false, Ordering::SeqCst);
// Also clear relay_active in case a previous relay task didn't run Drop.
// This is safe: if a relay task is genuinely active, it has its own guard and
// will re-acquire. If it crashed without clearing, this unblocks future /stream requests.
relay_active.store(false, Ordering::Release);  // relay_active needs to be Arc-cloned into run_connect scope
```

**Alternative simpler approach (no Arc threading):** Add a timeout health-check inside
`http_stream_server` — if `relay_active=true` for more than 60s with no new PCM data
delivered to `conn_tx`, force `relay_active=false`. This requires no changes to
`run_connect()` and keeps `relay_active` scoped entirely within `http_stream_server`.

The simpler approach is preferred. Inside the relay task loop (after line 656):
```rust
// Add a last_send_time Instant that updates on each successful conn_tx.send().
// In the main http_stream_server accept loop, a periodic health-check timer
// can reset relay_active if last_send_time > 60s ago.
```

**Alternative simplest fix:** Log relay state transitions. At the 503 relay-busy response
(line 574), add:
```rust
log::warn!("[spoton] /stream: relay-busy 503 — concurrent connection rejected");
```
This makes stuck `relay_active` visible in logs so it can be diagnosed and the correct
fix applied. For now, add the log and document the known limitation.

#### Fix 2 — Spirc reconnect notifies LMS ('ready' command)

**Location context:** `run_connect()` Spirc reconnect success branch (lines 1033-1039):
```rust
Ok((new_spirc, new_task)) => {
    {
        let mut guard = spirc_handle.lock().unwrap();
        *guard = Some(new_spirc);
    }
    current_spirc_task = Some(tokio::spawn(new_task));
    connecting = false;
}
```

**Pattern for notify()** (existing, lines 294-344):
```rust
async fn notify(&self, cmd: &str, p1: &str, p2: &str) {
    // ... HTTP/1.0 POST to LMS /jsonrpc.js spottyconnect command
    // p1 and p2 are the extra parameters
}
```

**Fix:** Call `lms.notify("ready", "", "")` after successful Spirc reconnect:
```rust
Ok((new_spirc, new_task)) => {
    {
        let mut guard = spirc_handle.lock().unwrap();
        *guard = Some(new_spirc);
    }
    current_spirc_task = Some(tokio::spawn(new_task));
    connecting = false;
    // Notify LMS that Spirc has reconnected — allows Perl to re-issue playlist play.
    lms.notify("ready", "", "").await;
    log::debug!("[spoton] Spirc reconnected — notified LMS 'ready'");
}
```

**lms variable scope:** `lms` is constructed at lines 888-893 and is in scope through the
main event loop. The reconnect branch is inside the same `loop {}` block, so `lms.notify()`
is directly callable. But the reconnect branch is in a `select!` arm — `lms` must be
accessible from that arm's scope. Verify this by checking if `lms` is moved into any closure
before the reconnect branch. Currently `lms` is moved into the LMS dispatcher task (line 896):
```rust
tokio::spawn(async move {
    ...
    lms.handle_player_event(&event, &mut current_track).await;
    ...
});
```
`lms` is moved into the spawned task. To call `lms.notify()` in the reconnect branch, the
`lms` value must either be cloned (LMS implements `Clone`, lines 115-131) or the notify must
be done via a channel. **Clone pattern:**
```rust
let lms_for_reconnect = lms.clone();  // before the `tokio::spawn(async move { lms ... })`
// Then in the reconnect branch:
lms_for_reconnect.notify("ready", "", "").await;
```

### `Plugins/SpotOn/Connect.pm` — add 'ready' command handler in `_connectEvent`

**Analog:** `Connect.pm` itself — existing command handler dispatch pattern (lines 450-718).

**Pattern: command handler dispatch** (existing structure in `_connectEvent`):
```perl
sub _connectEvent {
    my $request = shift;
    my $client  = $request->client();
    return unless defined $client;
    $client = $client->master;

    my $cmd = $request->getParam('_cmd');

    # Each command is an if-block. Return early to avoid fallthrough.
    if ($cmd eq 'start') { ... return; }
    if ($cmd eq 'change') { ... return; }
    if ($cmd eq 'stop') { ... return; }
    if ($cmd eq 'resume') { ... return; }
    if ($cmd eq 'volume') { ... return; }
    if ($cmd eq 'seek') { ... return; }

    main::INFOLOG && $log->is_info && $log->info("Unhandled spottyconnect command: $cmd");
}
```

**New handler to insert before the final `main::INFOLOG` line:**
```perl
# -----------------------------------------------------------------
# Ready: Spirc reconnected internally (after session expiry or source switch).
# Binary sends this after successfully completing a Spirc::new() reconnect.
# Re-issue playlist play so LMS resumes streaming without user intervention.
# -----------------------------------------------------------------
if ($cmd eq 'ready') {
    main::INFOLOG && $log->is_info && $log->info(
        "Spirc reconnected (ready event) for " . $client->id . " — re-issuing Connect play"
    );

    # Only re-issue if this player was previously the active Connect player.
    # If not (e.g. another player took over), ignore.
    if ($_activeConnectPlayer && $_activeConnectPlayer eq $client->id) {
        my $ts      = int(Time::HiRes::time() * 1000);
        my $playReq = $client->execute([
            'playlist', 'play',
            sprintf("spotify://connect-%u", $ts)
        ]);
        $playReq->source(__PACKAGE__);
    }
    return;
}
```

**Source-marking:** The `$playReq->source(__PACKAGE__)` pattern is used consistently in
existing `start` (line 631) and `resume` (line 561) handlers. Copy that pattern.

**Guard pattern:** Check `$_activeConnectPlayer` before acting (same pattern used in
`_onPause` lines 320-325 and `_onNewSong` lines 268-277).

## Shared Patterns

### JSON-RPC Notify Pattern (Binary → LMS)
**Source:** `librespot-spoton/src/connect.rs`, `notify()` method (lines 294-344)
**Apply to:** New 'ready' notify in `run_connect()` reconnect branch
```rust
// LMS JSON-RPC dispatch via HTTP/1.0 POST to /jsonrpc.js
// Command name is the spottyconnect _cmd parameter.
// p1 and p2 are the _p2 and _p3 parameters.
self.notify("ready", "", "").await;
```
Note: `notify()` takes `&self`, so it can be called on a cloned `LMS` value.
The clone is cheap — all fields are either `Arc<>` or `Option<String>` (cloning shares
the Arcs rather than duplicating the underlying data; see Clone impl lines 116-131).

**Source:** `Connect.pm`, `_onPause()` (lines 283-338) and `_onNewSong()` (lines 231-278)

### requestString / crackURL Pattern
**Source:** `/usr/share/perl5/Slim/Player/Protocols/HTTP.pm` (lines 921-984)
**Apply to:** `ProtocolHandler.pm::requestString` override

`crackURL` is the standard LMS URL parser used everywhere in the HTTP protocol stack.
Do not hand-roll URL parsing.

for fastest feedback loop:

1. **ProtocolHandler.pm** `canEnhanceHTTP` override — simplest, most likely to fix the

   Range header confusion.
3. **connect.rs** relay_active logging + reconnect 'ready' notify.
4. **Connect.pm** `ready` handler — only testable after step 3.
