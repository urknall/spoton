<objective>
Fix the architectural Device-ID mismatch between Discovery (mDNS) and Spirc (Spotify Cloud) in the Rust binary.

Purpose: `SessionConfig::default()` generates a random UUID as `device_id` on every call, while Discovery uses a deterministic FNV-1a hash of `cache_dir`. Spotify sees two different devices — Discovery announces one ID via mDNS, Spirc registers another with the Cloud. When a user connects via mDNS-discovered device, the credentials response targets the Discovery device_id, but Spirc does not recognize it. This is the root cause of the "device visible but connection fails" issue.

Output: Rebuilt `librespot-spoton` binary where both Discovery and Spirc share the same deterministic FNV-1a device_id.
</objective>
