# Phase 05.4 Plan 02: Per-Player Discovery Management Summary

**One-liner:** Per-player crash-loop protection with 30-minute cooldown timer, LMS-start reset, and Settings UI Discovery toggle â€” all backed by separate `discoveryDisabledByCrashLoop` pref to preserve user checkbox state.

## What Was Built

### Task 1: Daemon.pm â€” Crash-loop Protection + Cooldown Timer

Three focused changes to `Plugins/SpotOn/Connect/Daemon.pm`:

**A. `DISCOVERY_COOLDOWN_SECONDS` constant (1800 = 30 min, D-02)**
Added near the existing `MAX_FAILURES_BEFORE_DISABLE_DISCOVERY` constants.

**B. Rewritten `_checkStartTimes()` â€” per-player scope**
- Gets `$client` via `Slim::Player::Client::getClient($self->mac)`
- Checks if crash-loop is already active (`discoveryDisabledByCrashLoop` for $client, or `disableDiscovery` global as fallback)
- When crash-loop triggers and NOT already active: sets `$prefs->client($client)->set('discoveryDisabledByCrashLoop', 1)` (D-05: separate from user disableDiscovery checkbox)
- Schedules 30-minute cooldown timer via `Slim::Utils::Timers` (D-03)
- Fallback path: if no $client, sets global `disableDiscovery` (backward compat)

**C. New `_resetDiscoveryCooldown()` sub**
- Timer callback `(undef, $mac)` â€” Slim::Utils::Timers passes unused first arg
- Clears `discoveryDisabledByCrashLoop` per-player
- Calls `DaemonManager->stopHelper($client)` + `->startHelper($client)` to restart with discovery re-enabled

**D. Rewritten discovery flag evaluation in `start()`**
Three-source priority check (D-05: crash-loop flag overrides user checkbox):
1. `discoveryDisabledByCrashLoop` per-player (highest)
2. `disableDiscovery` per-player (user checkbox)
3. `disableDiscovery` global (fallback)

### Task 2: Five Supporting Files

**DaemonManager.pm:**
- `_isDiscoveryEnabled($client)` helper â€” mirrors `_isConnectEnabled` pattern, returns false if any of the three discovery-disable sources is truthy
- D-01 LMS-start reset loop in `init()`: iterates `Slim::Player::Client::clients()`, clears `discoveryDisabledByCrashLoop` for each player that has it set; also resets global `disableDiscovery` if set

**Plugin.pm:**
- `disableDiscovery => 0` added to `$prefs->init({...})` block (Pitfall 4: prevents undefined pref on first run)

**Settings.pm:**
- Per-player Discovery toggle save: `pref_enableDiscovery` coerced via `? 0 : 1` into `disableDiscovery` (T-05.4-03: input coercion, no arbitrary values stored). The existing `initHelpers()` call covers daemon restart.
- Template vars: `discoveryEnabled` (inverted disableDiscovery) and `discoveryByCrashLoop` added to per-player `if ($client)` block

**basic.html:**
- Discovery checkbox block inserted after Connect checkbox, before OGG override dropdown (inside `[% IF playerid %]` block)
- `[% IF discoveryByCrashLoop %]` warning div with orange text (PLUGIN_SPOTON_DISCOVERY_CRASH_LOOP_WARNING)
- Checkbox `disabled` when crash-loop active (D-05: user cannot override protection period)

**strings.txt:**
- 4 new PLUGIN_SPOTON_DISCOVERY keys with DE + EN translations
- ASCII-safe German (ae/oe/ue) per LMS convention
- Keys: `PLUGIN_SPOTON_DISCOVERY_ENABLED`, `_DESC`, `_LABEL`, `_CRASH_LOOP_WARNING`
