## Inline Fixes During UAT

### 1. Multi-player device_id collision (17b94bf)
All daemons shared the same cache_dir, producing identical FNV-1a device_ids. Spotify saw only one device. Fix: include player_mac in hash — `FNV-1a(cache_dir + player_mac)`.

### 2. mDNS announcing wrong IP (a5e58f4)
libmdns picked up 192.168.123.1 (virbr0) instead of 192.168.13.34 (wlp242s0). Fix: UDP socket default-route detection passed to `Discovery::builder().zeroconf_ip()`.

### 3. Volume normalization not passed to Connect daemons (22dd856)
`--enable-volume-normalisation` only injected into transcoding table, not Connect daemon args. Fix: one-line addition in Daemon.pm `start()`.
