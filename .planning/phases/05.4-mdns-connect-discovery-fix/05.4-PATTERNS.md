### `librespot-spoton/src/connect.rs` — Device-ID-Synchronisierung

**Analog:** Gleiche Datei, Zeilen 982-1001 (bestehender Discovery-Builder mit FNV-1a)

**Kernanliegen:** `session_config.device_id` wird aktuell als zufällige UUID via `SessionConfig::default()` erzeugt (Zeile 855). Discovery dagegen berechnet FNV-1a-Hash der `cache_dir` als Device-ID (Zeilen 988-991). Spirc und Discovery registrieren sich mit verschiedenen IDs bei Spotify. Fix: `device_id_shared` VOR Session und VOR Discovery berechnen, in beide propagieren.

**Bestehendes Discovery-Builder-Pattern** (Zeilen 982-1001 — kopieren und Device-ID herausziehen):
```rust
// AKTUELL (Zeile 988-991): FNV-1a device_id nur für Discovery berechnet
let device_id = {
    let mut h: u64 = 14695981039346656037;
    for b in cache_dir.as_bytes() { h ^= *b as u64; h = h.wrapping_mul(1099511628211); }
    format!("{:016x}", h)
};
const KEYMASTER_CLIENT_ID: &str = "65b708073fc0480ea92a077233ca87bd";
match librespot_discovery::Discovery::builder(device_id, KEYMASTER_CLIENT_ID.to_string())
    .name(device_name.to_string())
    .device_type(DeviceType::Speaker)
    .launch()
```

**Fix-Pattern: Device-ID vor Event-Loop berechnen** (neue Variable VOR Zeile 855 einfügen):
```rust
// NEU: Stabile device_id_shared — VOR session_config und Discovery (Zeile ~854)

    let mut h: u64 = 14695981039346656037;

    format!("{:016x}", h)

session_config.device_id = device_id_shared.clone();
```

**Bestehende `_checkStartTimes`-Funktion (zu ersetzen)** (Zeilen 191-214):

**Fix-Pattern: Per-Player + separates Crash-Loop-Flag + Cooldown-Timer**:
```perl
use constant DISCOVERY_COOLDOWN_SECONDS => 1800;  # D-02: 30 Minuten

sub _checkStartTimes {
    my $self = shift;
    my $client = Slim::Player::Client::getClient($self->mac);

    if ( scalar @{$self->_startTimes} >= MAX_FAILURES_BEFORE_DISABLE_DISCOVERY ) {
        splice @{$self->_startTimes}, 0,
               @{$self->_startTimes} - MAX_FAILURES_BEFORE_DISABLE_DISCOVERY;

        # Prüfe ob Crash-Loop noch nicht aktiv ist (verhindert Timer-Stack)
        my $alreadyDisabled = $client
            ? ($prefs->client($client)->get('discoveryDisabledByCrashLoop') || 0)
            : ($prefs->get('disableDiscovery') || 0);

        if ( time() - $self->_startTimes->[0] < MAX_INTERVAL_BEFORE_DISABLE_DISCOVERY
            && !$alreadyDisabled
        ) {
            $log->warn(sprintf(
                'SpotOn daemon crashed %s times within less than %s minutes - disabling discovery for %s min.',
                MAX_FAILURES_BEFORE_DISABLE_DISCOVERY,
                MAX_INTERVAL_BEFORE_DISABLE_DISCOVERY / 60,
                DISCOVERY_COOLDOWN_SECONDS / 60,
            ));

            if ($client) {
                # Per-Player: separates Flag (D-05: getrennt von User-Checkbox)
                $prefs->client($client)->set('discoveryDisabledByCrashLoop', 1);
            } else {
                $prefs->set('disableDiscovery', 1);
            }

            # Cooldown-Timer: Auto-Reset nach DISCOVERY_COOLDOWN_SECONDS (D-03)
            Slim::Utils::Timers::killTimers($self->mac, \&_resetDiscoveryCooldown);
            Slim::Utils::Timers::setTimer(
                $self->mac,
                Time::HiRes::time() + DISCOVERY_COOLDOWN_SECONDS,
                \&_resetDiscoveryCooldown,
                $self->mac,
            );
        }
    }
    push @{$self->_startTimes}, time();
}

sub _resetDiscoveryCooldown {
    my ($mac) = @_;
    my $client = Slim::Player::Client::getClient($mac) or return;

    $prefs->client($client)->set('discoveryDisabledByCrashLoop', 0);
    main::INFOLOG && $log->is_info && $log->info(
        "Discovery cooldown expired for $mac — re-enabling and restarting daemon"
    );

    require Plugins::SpotOn::Connect::DaemonManager;
    Plugins::SpotOn::Connect::DaemonManager->stopHelper($client);
    Plugins::SpotOn::Connect::DaemonManager->startHelper($client);
}
```

**Discovery-Flag-Übergabe anpassen** (Zeile 107 — per-Player statt global):
```perl
# AKTUELL (Zeile 107):
push @helperArgs, '--disable-discovery' if $prefs->get('disableDiscovery');

# NEU: Per-Player-Flag priorisieren, Crash-Loop-Flag hat höchste Priorität (D-05)
```

## Schlüsselarchitektur-Entscheidungen (für Planner)

1. **Fix-Reihenfolge:** (1) Device-ID-Fix in `connect.rs` (architektonischer Bug, klarer Fix), (2) Crash-Loop-Cooldown in `Daemon.pm`, (3) LMS-Start-Reset in `DaemonManager.pm`, (4) UI-Toggle in `Settings.pm` + `basic.html` + `strings.txt` + `Plugin.pm`. UAT nach Schritt 1+2 als Gate für Schritt 3+4.

2. **Neues Pref `discoveryDisabledByCrashLoop`** ist per-Player und von `disableDiscovery` (User-Checkbox) getrennt. `disableDiscovery` = Benutzerentscheidung, `discoveryDisabledByCrashLoop` = automatischer Schutz. Beide müssen in `Daemon.pm::start()` zusammen ausgewertet werden.

3. **Cooldown-Dauer: 30 Minuten** (D-02). `Slim::Utils::Timers::setTimer` mit `DISCOVERY_COOLDOWN_SECONDS = 1800`.

4. **Rust-Binary muss neu gebaut werden** nach `connect.rs`-Änderung: `cargo build --release` in `librespot-spoton/`. Planner muss dies als expliziten Task einplanen.

5. **UAT-Diagnose-Session (D-07)** ist eigenständiger Task, kein optionaler Schritt — Planner muss ihn als Gate vor den finalen UI-Tasks einplanen.
