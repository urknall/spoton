    - Package: `Plugins::SpotOn::DontStopTheMusic`
    - `use strict; use warnings;`
    - `use Digest::MD5 qw(md5_hex);`
    - `use Slim::Plugin::DontStopTheMusic::Plugin;`
    - `use Slim::Schema;`

    **init() sub:**
    - Calls `registerHandler('PLUGIN_SPOTON_RECOMMENDATIONS', \&dontStopTheMusic)`. The string key PLUGIN_SPOTON_RECOMMENDATIONS must exist in strings.txt (added in this plan's Task 2).

    **dontStopTheMusic($client, $cb) sub:**
    1. Get seed tracks: `Slim::Plugin::DontStopTheMusic::Plugin->getMixableProperties($client, 5)`.
    2. If no seed tracks or empty: `$cb->($client); return;`
    3. Get accountId: `$prefs->get('activeAccount') || ''`. If empty: `$cb->($client); return;`
    4. `require Plugins::SpotOn::API::Client;`
    5. Classify each seed track (Spotty-NG logic lines 56-73):
       - If `$track->{id}` is negative (RemoteTrack): look up via `Slim::Schema->find('Track', $track->{id})` and get the URL.
       - If `$track->{id}` matches `track:([a-z0-9]+)/i`: add to `$seedData->{seed_tracks}`.
       - Else if artist+title present: push to `@searchData` for search matching.
    6. Limit seed_tracks to max 5 entries.
    7. If `@searchData` exists: call `_searchForSeeds($client, $accountId, \@searchData, $seedData, $cb)` to match non-Spotify tracks.
    8. Else: call `_getRecommendations($client, $accountId, $seedData, $cb)` directly.

    **_searchForSeeds($client, $accountId, $searchDataRef, $seedData, $cb) sub:**
    Simplified version of Spotty-NG's nested search (SpotOn Client.pm doesn't have batch/series search). For each entry in @searchData:
    - Search: `Client->search($accountId, {q => sprintf('%s artist:"%s"', $title, $artist), type => 'track', limit => 5}, ...)`.
    - In callback: match track by title+artist (case-insensitive). If found, add to seed_tracks.
    - If not found by track: search `artist:"$artist"` with type => 'artist', limit => 5. Match by name, add to seed_artists.
    - Use a counter/remaining-ref pattern to chain the async searches. When all done: call `_getRecommendations`.
    - Limit total seeds (seed_tracks + seed_artists) to 5.

    **_getRecommendations($client, $accountId, $seedData, $cb) sub:**
    - If no seed_tracks and no seed_artists: `$cb->($client); return;`
    - Call `Plugins::SpotOn::API::Client->recommendations($accountId, $seedData, sub { ... })`.
    - In callback: if result is arrayref with tracks, map URIs to `spotify://track:XXX` format.
    - On empty/error result: call `_searchFallback` with first available seed artist.

    **_searchFallback($client, $accountId, $seedArtist, $cb) sub:**
    Per RESEARCH.md Pattern 5 and Claude's Discretion (DSTM Search-Fallback Details):
    - Random offset: `int(rand(40))`
    - `Client->search($accountId, {q => sprintf('artist:"%s"', $seedArtist), type => 'track', limit => 10, offset => $offset}, ...)`.
    - Map results to `spotify://track:XXX` URIs.
    - Call `$cb->($client, \@uris)`.
    - If no results: `$cb->($client)`.

    **URI mapping pattern** (Spotty-NG lines 43-49):
    - `$_->{uri} =~ /(track:.*)/; "spotify://$1"`

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-06 | Spoofing | DSTM seed data | accept | Seeds come from getMixableProperties() which reads the player's current playlist -- LMS-internal, no external input. |
| T-06-07 | Tampering | URI extraction from API response | mitigate | URI extracted via regex `/(track:[a-z0-9]+)/i` -- only alphanumeric track IDs pass. Injected URIs that don't match pattern are silently dropped. |
| T-06-08 | Information Disclosure | recommendations API | accept | Seed tracks/artists sent to Spotify API are already known to Spotify (user's own playlist data). No new data exposure. |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan. |
