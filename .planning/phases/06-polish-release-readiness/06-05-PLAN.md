## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo.xml | Static file served via GitHub raw URL -- integrity relies on GitHub access control |
| install.xml | Static metadata -- no runtime trust boundary |
| LMS repository download | LMS verifies SHA1 of downloaded zip against repo.xml sha attribute |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-11 | Tampering | repo.xml on GitHub | mitigate | GitHub repository access control. SHA1 verification by LMS on download. Release process must compute fresh SHA1 per release. |
| T-06-12 | Tampering | Plugin zip download | mitigate | LMS verifies SHA1 from repo.xml against downloaded zip. HTTPS transport from GitHub. |
| T-06-13 | Elevation | install.xml defaultState=enabled | accept | Plugin auto-enables on install -- standard LMS behavior for all plugins. User must explicitly add repository URL first. |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan. |
</threat_model>
