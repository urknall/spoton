  <name>Task 1: Create Cross.toml and build all 5 Linux musl-static binaries</name>
  <files>librespot-spoton/Cross.toml, Plugins/SpotOn/Bin/x86_64-linux/spoton, Plugins/SpotOn/Bin/aarch64-linux/spoton, Plugins/SpotOn/Bin/armhf-linux/spoton, Plugins/SpotOn/Bin/arm-linux/spoton, Plugins/SpotOn/Bin/i386-linux/spoton</files>
  <action>
All work runs from the librespot-spoton/ directory.

Step 1 - Create Cross.toml:

Create librespot-spoton/Cross.toml with no custom image overrides (the default cross-rs images for musl targets already include musl-tools). The file documents the supported targets:

```
[target.x86_64-unknown-linux-musl]
[target.aarch64-unknown-linux-musl]
[target.armv7-unknown-linux-musleabihf]
[target.arm-unknown-linux-musleabihf]
[target.i686-unknown-linux-musl]
```

No image overrides needed -- default cross-rs images for these targets include musl-tools and work with rustls-tls (per D-05, no OpenSSL dependency).

Step 2 - Build each Linux target sequentially (per D-01, all local via cross-rs Docker):

Build order chosen for fastest feedback -- x86_64 first (confirms build config), then ARM targets, then i686:

1. `cross build --release --target x86_64-unknown-linux-musl`
   Copy target/x86_64-unknown-linux-musl/release/spoton to Plugins/SpotOn/Bin/x86_64-linux/spoton
   This REPLACES the existing glibc-linked binary (ARCH-01). Remove the old binary first.

2. `cross build --release --target aarch64-unknown-linux-musl`
   Copy target/aarch64-unknown-linux-musl/release/spoton to Plugins/SpotOn/Bin/aarch64-linux/spoton
   Remove .gitkeep from that directory.

3. `cross build --release --target armv7-unknown-linux-musleabihf`
   Copy target/armv7-unknown-linux-musleabihf/release/spoton to Plugins/SpotOn/Bin/armhf-linux/spoton
   Remove .gitkeep from that directory.

4. `cross build --release --target arm-unknown-linux-musleabihf`
   Copy target/arm-unknown-linux-musleabihf/release/spoton to Plugins/SpotOn/Bin/arm-linux/spoton
   Remove .gitkeep from that directory.

5. `cross build --release --target i686-unknown-linux-musl`
   Copy target/i686-unknown-linux-musl/release/spoton to Plugins/SpotOn/Bin/i386-linux/spoton
   Remove .gitkeep from that directory.

Step 3 - Verify each Linux binary:

For each of the 5 binaries:
- Run `file Plugins/SpotOn/Bin/{dir}/spoton` to confirm ELF type and architecture
- Run `ldd Plugins/SpotOn/Bin/{dir}/spoton` -- must report "not a dynamic executable" (per D-04, musl-static)
- The x86_64 binary can be executed directly: `Plugins/SpotOn/Bin/x86_64-linux/spoton -n Test --check` must output "ok spoton v1.0.0" followed by JSON capabilities

Expected `file` output patterns:
- x86_64-linux: "ELF 64-bit LSB ... x86-64 ... statically linked"
- aarch64-linux: "ELF 64-bit LSB ... ARM aarch64 ... statically linked"
- armhf-linux: "ELF 32-bit LSB ... ARM ... statically linked"
- arm-linux: "ELF 32-bit LSB ... ARM ... statically linked"
- i386-linux: "ELF 32-bit LSB ... Intel 80386 ... statically linked"

IMPORTANT: Each cross build takes 5-15 minutes depending on target. Total build time ~30-60 minutes. Use --release flag for all builds (production binaries). Do NOT use --jobs to limit parallelism inside Docker -- cross-rs handles this.
