  <name>Task 1: Extend Helper.pm init() for Windows and clean up _findBin() dead code</name>
  <files>Plugins/SpotOn/Helper.pm</files>
  <action>
Two changes to Helper.pm:

CHANGE 1 - Extend init() to add Windows binary path (per D-10):

The current init() only adds the aarch64->armhf fallback. Add a Windows clause.

After the existing aarch64 block (line 22-27), add a Windows block:

```
# Windows: add x86_64-win64 binary directory to search path
# LMS initSearchPath does not map Windows archname to x86_64-win64
if ( main::ISWINDOWS ) {
    Slim::Utils::Misc::addFindBinPaths(
        catdir(Plugins::SpotOn::Plugin->_pluginDataFor('basedir'), 'Bin', 'x86_64-win64')
    );
}
```

Per D-12, the ARM fallback chain is: aarch64 -> armhf, armv7 -> arm. The aarch64 -> armhf fallback already exists in init() (line 22-27). The armv7 -> arm fallback is already handled by LMS initSearchPath (it pushes arm-linux after armhf-linux for armhf-linux archname). No additional fallback paths needed.

Per D-13, spoton-custom override remains first in the candidate list -- this is already handled in _findBin() line 130 and must NOT be changed.

CHANGE 2 - Remove dead code from _findBin() (per PATTERNS.md analysis):

Lines 122-127 in _findBin() push `HELPER . '-x86_64'` as a candidate binary name. This searches for a binary named "spoton-x86_64" which does not exist anywhere. LMS already registers Bin/x86_64-linux/ first in @findBinPaths via initSearchPath, so findbin('spoton') already finds the correct x86_64 binary.

Remove the entire block:
```perl
    if (Slim::Utils::OSDetect::OS() eq 'unix') {
        # on 64-bit x86, try the x86_64 build first
        if ( $Config::Config{'archname'} =~ /x86_64/ ) {
            push @candidates, HELPER . '-x86_64';
        }
    }
```

After removal, _findBin() becomes:
```perl
sub _findBin {
    my ($checkerCb, $customFirst) = @_;

    my @candidates = (HELPER);    # 'spoton'
    my $binary;

    # Custom override first (LMS-10 preparation, per D-13)
    unshift @candidates, HELPER . '-custom';

    foreach my $name (@candidates) {
        ...
    }
    ...
}
```

The Config import at line 5 (`use Config;`) can also be removed since $Config::Config{'archname'} is no longer referenced anywhere in the file. The aarch64 detection in init() uses Slim::Utils::OSDetect::details()->{osArch}, not Config.

Do NOT touch any other code. The public API (get, helperCheck, getCapability, getVersion, version) is unchanged. All consumers (DaemonManager, ProtocolHandler, Plugin.pm, Settings.pm) call Helper->get() and are binary-path-agnostic.
