  All commands run via SSH to the Pi at 192.168.13.5 as user `sti`.

  **Part A: Install SpotOn via LMS custom repository**

  1. Add the SpotOn repo URL to LMS custom repositories. Use the LMS JSON-RPC API:
     ```
     ssh sti@192.168.13.5 "curl -s -X POST http://localhost:9000/jsonrpc.js \
       -d '{\"id\":1,\"method\":\"slim.request\",\"params\":[\"\",[\"pref\",\"plugin_repo\",\"?\"]]}'
     ```
     This retrieves the current list of additional repositories. Then append the SpotOn repo URL:
     `https://raw.githubusercontent.com/stiefenm/spoton/main/repo.xml`

     If direct pref manipulation is complex, provide manual instructions as fallback:
     LMS Settings -> Plugins -> Additional Repositories -> add the URL -> Apply.

     Alternative automated approach: Edit the LMS prefs file directly:
     ```
     ssh sti@192.168.13.5 "sudo grep -q 'stiefenm/spoton' /var/lib/squeezeboxserver/prefs/server.prefs || echo 'The repo URL needs to be added manually via LMS UI'"
     ```

  2. Trigger LMS plugin update check or restart LMS to pick up the new repo:
     ```
     ssh sti@192.168.13.5 "sudo systemctl restart lyrion"
     ```

  3. After LMS restarts, install SpotOn via JSON-RPC or verify it appears in the plugin list. The simplest reliable approach is:
     - Add the repo URL via LMS web UI (http://192.168.13.5:9000/settings/index.html -> Plugins tab -> Additional Repositories)
     - Enable SpotOn in the plugin list
     - Apply and restart

     If CLI installation is possible:
     ```
     ssh sti@192.168.13.5 "curl -s -X POST http://localhost:9000/jsonrpc.js \
       -d '{\"id\":1,\"method\":\"slim.request\",\"params\":[\"\",[\"pref\",\"set\",\"plugin_repo\",\"https://raw.githubusercontent.com/stiefenm/spoton/main/repo.xml\"]]}'"
     ```

     Note: LMS plugin installation via CLI may require user interaction in the web UI. If automated installation is not straightforward, this becomes a checkpoint step. The executor should attempt automation first, then fall back to manual if needed.

  4. After installation, verify SpotOn appears in the installed plugins:
     ```
     ssh sti@192.168.13.5 "ls /var/lib/squeezeboxserver/cache/InstalledPlugins/Plugins/SpotOn/ 2>/dev/null && echo 'SpotOn installed'"
     ```

  5. Verify SpotOn loads in LMS by checking the log:
     ```
     ssh sti@192.168.13.5 "grep 'plugin.spoton' /var/log/squeezeboxserver/server.log | tail -5"
     ```

  **Part B: Create SpotOn monitoring script and cron job**

  6. Create the monitoring script on the Pi at `/usr/local/bin/spoton-monitor.sh`. This is adapted from the existing Spotty monitor at `/usr/local/bin/spotty-monitor.sh` (same offset-tracking architecture, updated grep patterns for SpotOn).

     The script content:
     ```bash
     #!/bin/bash
     # SpotOn production monitor — extracts plugin-relevant log entries
     # Runs via cron every 30 min, appends to daily log file

     LOGFILE="/var/log/squeezeboxserver/server.log"
     OUTDIR="/var/log/squeezeboxserver/spoton-monitor"
     TODAY=$(date +%Y-%m-%d)
     OUTFILE="$OUTDIR/spoton-$TODAY.log"
     STAMPFILE="$OUTDIR/.last-offset"

     mkdir -p "$OUTDIR"

     # Track where we left off to avoid duplicate lines
     OFFSET=0
     [ -f "$STAMPFILE" ] && OFFSET=$(cat "$STAMPFILE")
     CURRENT_SIZE=$(wc -c < "$LOGFILE" 2>/dev/null || echo 0)

     # Log was truncated/rotated — reset offset
     if [ "$CURRENT_SIZE" -lt "$OFFSET" ]; then
         OFFSET=0
     fi

     if [ "$CURRENT_SIZE" -gt "$OFFSET" ]; then
         tail -c +$((OFFSET + 1)) "$LOGFILE" | grep -iE \
             "plugin\.spoton|SpotOn|_connectEvent|_onNewSong|_onPause|Connect.*start|Connect.*stop|stream_port|sync.*group|error.*spoton|FATAL|segfault|503|429|stream failed|proxy.*redirect|spirc|newmetadata|getNextTrack.*no-op|librespot|spoton-daemon|ZeroConf|tokenRefresh" \
             | grep -v "getMetadataFor" \
             >> "$OUTFILE" 2>/dev/null
     fi

     echo "$CURRENT_SIZE" > "$STAMPFILE"

     # Clean up logs older than 30 days
     find "$OUTDIR" -name "spoton-*.log" -mtime +30 -delete 2>/dev/null
     ```

     Write the script via SSH:
     ```
     ssh sti@192.168.13.5 "cat > /tmp/spoton-monitor.sh << 'SCRIPT'
     ... (script content above)
     SCRIPT
     sudo mv /tmp/spoton-monitor.sh /usr/local/bin/spoton-monitor.sh
     sudo chmod +x /usr/local/bin/spoton-monitor.sh"
     ```

  7. Create the cron job:
     ```
     ssh sti@192.168.13.5 "echo '*/30 * * * * root /usr/local/bin/spoton-monitor.sh' | sudo tee /etc/cron.d/spoton-monitor"
     ```

  8. Create the log directory:
     ```
     ssh sti@192.168.13.5 "sudo mkdir -p /var/log/squeezeboxserver/spoton-monitor && sudo chown squeezeboxserver:squeezeboxserver /var/log/squeezeboxserver/spoton-monitor"
     ```
     Note: The cron runs as root, so ownership is not strictly necessary, but keeping it consistent with other LMS log dirs is good practice.

  9. Test the monitor by running it once:
     ```
     ssh sti@192.168.13.5 "sudo /usr/local/bin/spoton-monitor.sh && ls -la /var/log/squeezeboxserver/spoton-monitor/"
     ```

      ```

      ```
