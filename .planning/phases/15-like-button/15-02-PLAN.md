## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LMS TrackInfo -> trackInfoMenu | $remoteMeta->{uri} from LMS framework — could contain arbitrary strings for non-SpotOn tracks |
| Plugin -> Client.pm -> Spotify API | API responses may contain unexpected structures or error codes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | trackInfoMenu $remoteMeta->{uri} | mitigate | Regex validation: $trackUri =~ /^spotify:track:[A-Za-z0-9]+$/ — rejects injection, non-SpotOn URIs, and malformed input before any API call or cache key construction |
| T-15-02 | Info Disclosure | Cache key leaking account info | accept | Cache keys contain accountId (opaque hash, not PII) and trackId (public Spotify ID). No sensitive data exposed. Cache is server-local only |
| T-15-03 | Denial of Service | Repeated checkTracks API calls | mitigate | D-07: 60s TTL cache. D-06: on-demand only (no pre-fetch). Single URI per call. Rate limiting via Client.pm _request |
| T-15-04 | Spoofing | Stale liked state in cache | accept | 60s TTL is acceptable staleness window. D-08: immediate invalidation after user action eliminates most staleness. Worst case: menu shows wrong state for up to 60s if external change |
| T-15-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed — all modules bundled with LMS |
</threat_model>
