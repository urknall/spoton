## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User search query -> Spotify API | Search text passed to API::Client::search which URL-encodes via SimpleAsyncHTTP |
| Show/Episode IDs from API response -> ProtocolHandler | IDs come from trusted Spotify API response, not user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-02 | Tampering | _showFeed showId | mitigate | showId comes from _showItem passthrough (API response data), not user input; ID validation guard exists in Client.pm getShowEpisodes (Phase 18 T-18-01) |
| T-19-03 | Tampering | _podcastSearchFeed query | mitigate | Query passed through Client.pm search which URL-encodes via SimpleAsyncHTTP; no direct URL interpolation in Plugin.pm |
| T-19-04 | Information Disclosure | _episodeItem metadata cache | accept | Cache stores only public metadata (title, show name, artwork URL); same pattern as _trackItem; TTL 7 days |
| T-19-05 | Spoofing | spoton://episode:ID URI | accept | URI constructed from Spotify API response data (trusted source); ProtocolHandler validates URI format before spawning binary |
</threat_model>
