B) In ProtocolHandler.pm, add an explodePlaylist class method. Place it after the getNextTrack sub (after line 299) and before parseDirectHeaders. The method resolves spoton:// URIs for containers into lists of playable track/episode URLs.

Method signature: sub explodePlaylist { my ($class, $client, $uri, $cb) = @_; }

Logic:
- If $uri matches spoton://track:([A-Za-z0-9]+) — single track, return [$uri] immediately via $cb
- If $uri matches spoton://episode:([A-Za-z0-9]+) — single episode, return [$uri] immediately via $cb
- If $uri matches spoton://album:([A-Za-z0-9]+) — fetch ALL tracks via recursive page fetch:
  Resolve accountId via Plugins::SpotOn::Plugin::_getAccountId($client). Use require Plugins::SpotOn::API::Client. Implement a helper closure _fetchAllAlbumTracks that fetches getAlbumTracks with offset=0, limit=50, collects track items, and if total > fetched, recurse with offset+=50. When all pages fetched, map each track item to 'spoton://track:' . $track->{id} and pass the arrayref to $cb. If API returns undef, call $cb->([]).
- If $uri matches spoton://playlist:([A-Za-z0-9]+) — fetch ALL tracks via recursive page fetch:
  Same pattern as album but using getPlaylistItems with limit=100. Playlist items are wrapped: $_->{track}{id}. Skip items where track is undef (local files). Map to 'spoton://track:' . $track_id.
- If $uri matches spoton://show:([A-Za-z0-9]+) — fetch first page of episodes (limit=50, no full pagination needed):
  Use getShowEpisodes. Map to 'spoton://episode:' . $episode->{id}. Pass to $cb.
- Default (no match): call $cb->([$uri]) — pass through unchanged

Add logging: main::INFOLOG and $log->is_info for each explode case with the URI and result count.
