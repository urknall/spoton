<!-- LMS CSRF module — key method for raw function handlers -->

From /usr/share/perl5/Slim/Web/HTTP/CSRF.pm:
  isRequestCSRFSafe($class, $request, $response, $params, $providedPageAntiCSRFToken)
  - Returns 1 if request has valid X-Requested-With: XMLHttpRequest header
  - Returns 1 if request has valid Referer matching Host
  - Returns 1 if valid cauth token present
  - Returns 0 otherwise

The simplest check for AJAX endpoints: verify $response->request->header('X-Requested-With') eq 'XMLHttpRequest'.
This is exactly what LMS's own isRequestCSRFSafe checks first (line 111).
Checking csrfProtectionLevel pref ensures the guard is only active when LMS has CSRF protection enabled.
</interfaces>

Three files to modify:

**Settings.pm — Add _csrfCheck helper and guard the three write handlers:**

1. Add a private helper sub _csrfCheck near the other helper subs (after _isDiscoveryRunning, before the closing 1;):

   sub _csrfCheck {
       my ($httpClient, $response) = @_;
       my $serverPrefs = preferences('server');
       return 1 unless $serverPrefs->get('csrfProtectionLevel');
       my $request = $response->request;
       return 1 if $request
           && $request->header('X-Requested-With')
           && $request->header('X-Requested-With') eq 'XMLHttpRequest';
       $response->code(403);
       $response->header('Content-Length' => 0);
       $response->header('Connection' => 'close');
       $response->content_type('text/plain');
       Slim::Web::HTTP::addHTTPResponse($httpClient, $response, \'');
       return 0;
   }

   Logic: if csrfProtectionLevel is 0 or unset, skip check (CSRF disabled). Otherwise, require X-Requested-With: XMLHttpRequest. Return 403 if missing.

2. Add CSRF guard as the first line inside each of these three handlers:
   - _clearLogsHandler: add "return unless _csrfCheck($httpClient, $response);" as first line after parameter unpacking
   - _discoveryStartHandler: same
   - _discoveryStopHandler: same

   Do NOT add the guard to _discoveryStatusHandler (read-only, polling endpoint) or _diagnosticBundleHandler (read-only, already gated by diagnosticMode pref).

**basic.html — Add X-Requested-With header to all three AJAX calls:**

1. In spotonDiscovery() function (around line 205): after xhr.open(), add:
   xhr.setRequestHeader('X-Requested-With', 'XMLHttpRequest');

2. In spotonClearLogs() function (around line 220): after xhr.open(), add:
   xhr.setRequestHeader('X-Requested-With', 'XMLHttpRequest');

Note: The discoveryStatus polling XHR (line 13) is read-only and unprotected, so no header needed there. But adding it is harmless and consistent — add it there too for defense-in-depth.
