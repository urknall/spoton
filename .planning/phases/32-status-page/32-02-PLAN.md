    - var pollTimer;
    - function poll(): XHR GET to /plugins/SpotOn/status/data. On 200: parse JSON, call render(data), hide error banner, flash poll indicator (opacity 1 then back to 0.7 via setTimeout 200ms), update last-updated timestamp. On non-200 or network error: show error banner, keep polling.

Add hr separator between multiple daemons.

    API card: metric rows for Total Requests (apiRequestCount), In-flight (inflightCount), 429 Responses (api429Count), Rate Limited Own (Yes in rate-limited class or No), Rate Limited Bundled (same), Active Accounts (accountCount), Discovery Running (Yes/No).

    Errors card: clear errors-body. If data.errors.length === 0, set "No errors recorded." muted. Otherwise for each error (already newest-first from backend): create error-row div with appropriate level class, timestamp formatted via new Date(e.ts * 1000).toLocaleTimeString(), module in brackets with mono class, message via textContent with title attribute for full text (truncate display at 80 chars using substring).

    System card (D-03 — static data, not re-rendered after first load): Render metric rows for Plugin Version (with v prefix), Binary Version (with v prefix), LMS Version, Perl Version, OS. Set footer text to include version. Use a flag var systemLoaded = false; only render system card content once (first poll).

    Uptime formatter function: takes seconds, returns string. If s is 0 or null, return em-dash. Otherwise: days = Math.floor(s/86400), hours = Math.floor((s%86400)/3600), minutes = Math.floor((s%3600)/60). Return formatted string, omitting 0d prefix if days is 0.

    Null/undefined display: values that are null or undefined show em-dash character. Zero values show "0".
