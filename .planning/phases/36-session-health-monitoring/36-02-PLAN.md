<objective>
Add health-aware daemon monitoring to Perl: periodic async health checks that detect and restart daemons with stale Spotify sessions, clean up misleading watchdog log noise, and surface session health on the Status Page.

Purpose: The enhanced Rust /health endpoint (Plan 01) provides session state data. This plan makes the Perl side act on it — restarting daemons before users hit cold-start playback failure — and displays the health data on the Status Page for diagnostic visibility.

Output: Modified Daemon.pm (accessors), DaemonManager.pm (health check infrastructure + log cleanup), Status.pm (daemon health data), status.html (health display).
</objective>

        ```
                Slim::Networking::SimpleAsyncHTTP->new(
                    sub { $class->_onHealthResponse($helper, @_) },

            }
        }
        ```
        Note: The `$helper->alive` check is redundant with the outer if/elsif but makes the block self-documenting. The `_streamPort` check prevents requests to daemons that haven't announced their port yet.

    B3. Add three new methods to DaemonManager.pm, BEFORE the `stopHelper` sub (line 339). Insert them after line 228 (end of _streamAlivePoll):

    METHOD: `_onHealthResponse($class, $helper, $http)`:
    - Guard: `return unless $helper && $helper->alive;` (edge case: daemon restarted between request and callback)
    - Parse JSON: `my $json = eval { from_json($http->content) };`
    - Store health data on daemon: `$helper->_lastHealthSession({ session_valid => ..., session_age_secs => ..., idle_secs => ..., checked_at => time() })` (always store, even before restart checks)
    - Malformed response check: unless ($json && defined $json->{status} && $json->{status} eq 'ok'), call `_restartForHealth($helper, 'malformed health response')` and return
    - Signal 1: if (!$json->{session_valid}), call `_restartForHealth($helper, 'session_valid=false')` and return
    - Signal 2: if ($json->{session_age_secs} > 14400 && $json->{idle_secs} > 300), call `_restartForHealth($helper, sprintf('stale session (age=%ds, idle=%ds)', $json->{session_age_secs}, $json->{idle_secs}))` and return

    METHOD: `_onHealthError($class, $helper, $http)`:
    - Log at warn level: "Health check failed for {mac}: {error}"
    - Do NOT restart (avoid double-restart race with _streamAlivePoll alive check)

    METHOD: `_restartForHealth($class, $helper, $reason)`:
    - Guard: `return unless $helper && $helper->alive;`
    - Log at INFO: `"Health check restart for {mac}: {reason}"`
    - Reset health check counter: `$helper->_healthCheckCount(0);`
    - Call `$class->stopHelper($helper->mac);`
    - Call `$class->startHelper($helper->mac);`
