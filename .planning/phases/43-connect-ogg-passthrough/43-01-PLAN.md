  <action>
Two changes in one crate, both required for Connect OGG audio to work.

**A. connect.rs — HttpStreamSink::write() AudioPacket::Raw handling (D-01)**

Note: HttpStreamSink is NOT the production Connect sink (UnifiedHttpStreamSink in unified.rs already handles AudioPacket::Raw since Phase 42). This fix is for code consistency in the standalone sink.

In HttpStreamSink::write() (around line 401), replace the irrefutable let-else pattern that discards AudioPacket::Raw with an explicit match, following the same pattern already in UnifiedHttpStreamSink::write() and BrowseHttpSink::write():
- The Samples arm retains ALL existing logic unchanged: f64_to_s16 conversion, wall-clock rate-limiter with frames_consumed/SAMPLE_RATE/buffer_latency_ns math, try_send loop.
- The Raw arm forwards bytes via Bytes::copy_from_slice with backpressure-only pacing: a try_send loop with 1ms sleep on Full and SinkError on Closed. No wall-clock rate-limiting (per D-02, OGG pages have variable size; backpressure-only pacing matches BrowseHttpSink).
- Move the final Ok(()) to be the return of the match expression.

**B. unified.rs — OGG header buffering in UnifiedHttpStreamSink (D-03)**

The production Connect sink (UnifiedHttpStreamSink) already forwards AudioPacket::Raw. The problem is that the /stream handler (around line 403-408) drains ALL queued chunks on new connections, discarding OGG BOS+Vorbis headers that the decoder needs. Per D-03, buffer OGG headers and replay them on each /stream connection.

B1. Add fields to UnifiedHttpStreamSink struct:
- ogg_header_buf of type Arc of std::sync::Mutex of Vec of Bytes — shared with the HTTP handler for replay
- collecting_headers of type bool — tracks whether we are still in the header portion of a track

B2. Extend UnifiedHttpStreamSink::open() signature:
- Add ogg_header_buf parameter (same Arc type as the struct field)
- Add passthrough parameter (bool) — used to initialize collecting_headers
- Store both in the struct. Initialize collecting_headers to the passthrough value (only collect headers when passthrough is active).

B3. Modify UnifiedHttpStreamSink start():
- Lock ogg_header_buf, call clear() on it, drop the lock
- Set collecting_headers back to true (if passthrough is active — check if ogg_header_buf is non-empty Arc, which implies passthrough mode)

Actually, simpler: store the passthrough bool as a field on the sink (add it). In start(), if self.passthrough, clear the buffer and set collecting_headers = true.

B4. Modify UnifiedHttpStreamSink write() in the AudioPacket::Raw arm:
- Before sending to the channel, if self.collecting_headers is true:
  - Check if the chunk is an OGG header page: bytes length >= 27, first 4 bytes equal b"OggS", and bytes 6 through 13 are all zero (granule_position == 0 indicates header page in Vorbis)
  - If header page: lock ogg_header_buf, push a clone of the chunk bytes
  - If NOT a header page (granule_position != 0, meaning first audio data page): set self.collecting_headers = false. Do NOT buffer this chunk.
- Always send the chunk to the channel via try_send as currently done (the channel data is the live stream; the buffer is the replay copy)

B5. Thread the ogg_header_buf Arc through run_unified() to unified_http_server():
- In run_unified(), inside the enable_connect block (around line 970-1005):
  - Create the Arc: let ogg_header_buf = Arc::new(std::sync::Mutex::new(Vec::new()));
  - Clone it for the sink closure: let ogg_buf_for_sink = Arc::clone(ogg_header_buf);
  - Pass ogg_buf_for_sink and passthrough into UnifiedHttpStreamSink::open() inside the move closure
- Store ogg_header_buf as Option<Arc<...>> alongside pcm_rx_arc and flush_rx_arc
- Add the ogg_header_buf parameter to unified_http_server() signature (as Option<Arc<std::sync::Mutex<Vec<Bytes>>>>), following the pcm_rx pattern
- At call site 1 (line 1171, Connect mode): pass Some(ogg_header_buf_arc)
- At call site 2 (line 1476, Browse-only mode): pass None

B6. Modify /stream handler to replay OGG headers after drain:
- After the drain loop (around line 403-408), add a header replay block guarded by passthrough:
  - If passthrough is true, lock the ogg_header_buf (unwrap from the Option), clone the Vec contents, drop the lock
  - For each buffered header chunk, send it to conn_tx via try_send (this is the per-connection relay channel created at line 412). On Full, sleep 1ms and retry. On Closed, log warning and break.
  - Log the count of replayed headers at debug level: "[spoton/unified] /stream: replayed N OGG header chunks"
- This ensures the decoder receives BOS + Vorbis headers before any audio data, even though the drain removed them from the main channel.
