<objective>
Add granule_position-based wall-clock rate-limiting to the Connect passthrough OGG path in both
UnifiedHttpStreamSink (production) and HttpStreamSink (standalone), fixing the Spirc/audio desync
where the decoder races through compressed OGG data in seconds instead of real-time. Per D-01,
the granule_position from the OGG page header replaces frames_consumed as the time reference in
the CON-14 formula. Per D-04/D-05, BrowseHttpSink is untouched.

Purpose: Without rate-limiting, OGG passthrough fills the 5MB buffer chain in seconds, causing
Spirc to fire TrackChanged/EndOfTrack far ahead of actual audio playback. Pause, skip, and
progress from the Spotify app all target the wrong track. This fix paces OGG delivery at
wall-clock speed, identical to how PCM is already paced.

Output: Modified unified.rs and connect.rs with rate-limited OGG Raw arms.
</objective>

    Modify the AudioPacket::Raw arm in UnifiedHttpStreamSink::write() (unified.rs) to add
    granule_position-based rate-limiting per D-01, D-02, D-03. The change goes AFTER the
    existing collecting_headers block and BEFORE the existing try_send loop.

    The rate-limiting logic parses the granule_position as an i64 little-endian from
    chunk bytes 6 through 14 (exclusive), guarded by a length check (chunk.len() >= 14)
    and an OggS magic check (chunk[0..4] == b"OggS"). Per D-03, if the granule_position
    is less than or equal to zero (header pages), no sleep occurs. Per D-01, when the
    granule_position is positive (audio pages), apply the CON-14 formula:

      expected_ns = (granule_position as u128) * 1_000_000_000 / SAMPLE_RATE as u128
                    + self.buffer_latency_ns
      elapsed_ns  = self.began_at.elapsed().as_nanos()
      if expected_ns > elapsed_ns: sleep the difference

    This is the same formula shape as the Samples arm, with granule_position replacing
    self.frames_consumed. The i64-to-u128 cast is safe because granule_position is always
    non-negative for audio pages (we checked > 0). Use try_into().expect() for the 8-byte
    slice conversion to [u8; 8], matching the codebase's established panic-on-logic-error
    pattern for Ogg page parsing.

    Per D-07, do NOT remove or modify self.frames_consumed — it remains for PCM mode.

    Apply the identical rate-limiting logic to the AudioPacket::Raw arm in
    HttpStreamSink::write() (connect.rs), between the chunk creation and the try_send
    loop. HttpStreamSink already has began_at and buffer_latency_ns fields. Per D-05,
    this is for code consistency with the production sink.

    Replace the existing Phase 43 comment in the connect.rs Raw arm that says
